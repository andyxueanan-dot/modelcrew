# ModelCrew 演示视频 · 逐镜脚本（目标 75 秒）

> 用途：QoderWork CN 大赛·赛道八「多 Agent 编排」论坛帖配套录屏。
> 编排赛**动态演示 >> 静态截图**——评委要看的是"团队真的被自动召唤、Critic 真的拦了一个错误结论"。
> 录制：屏幕录制（OBS / Win+G / ScreenToGif 均可），1080p，**全程不需要露脸不需要配音**——按"画外字幕/旁白文案"打字幕即可。
> 总时长控制在 **60–90 秒**（论坛/评委注意力窗口）。

---

## 一句话主线（视频要讲的故事）
"我对一支可复用的 AI 团队说一句话 → 它自动召唤 7 个角色分工 → 最关键的是那个**反幻觉 Critic**，把'网球动量真实存在'这个最诱人的结论判成了 ❌（热手谬误）。"

## 分镜表

| # | 时间 | 画面（录什么） | 字幕/旁白文案（打在画面上） |
|---|------|----------------|------------------------------|
| 1 | 0–8s | Qoder 界面，输入框打字：`@数学建模 ModelCrew`，弹出套件 | 「这是我做的可复用建模 AI 团队 ModelCrew，一个专家套件 = 7 个角色」 |
| 2 | 8–18s | 继续输入任务：「按 2024 MCM C 题路由：网球逐分数据，判断'动量'是不是真的」并回车 | 「我只说一句话，不指定用哪个 Agent」 |
| 3 | 18–30s | Router 输出：判为**数据型(C)**，给出召唤链路 Analyst→Scout→Modeler→Solver→Critic→Writer + 时序图 | 「调度中枢自动判题型、**自动召唤**对应专家、排好协作顺序」 |
| 4 | 30–42s | 快速滚过 Scout/Modeler/Solver 的产出；Solver 真跑 `solve.py` 出 runs test / 置换检验 / AUC | 「数据侦察→建模→求解，Solver 真跑代码、不编数字（seed=42 可复现）」 |
| 5 | 42–60s | ⭐**高光**：停在 Critic 审计页，放大 **C2「动量真实存在」→ ❌ 证伪** 这一行；展示三方辩论(理论家/怀疑者/裁判) | 「最关键的一步——反幻觉 Critic 对抗式审计：把最诱人的'动量真实'判为 ❌，这是统计学经典的**热手谬误**」 |
| 6 | 60–70s | Writer 出论文，结论"数据不支持非随机动量"，局限性如实写入 | 「写手把 Critic 的结论如实写进论文，诚实反而加分」 |
| 7 | 70–75s | 切到一行字幕 + GitHub 仓库地址 | 「同一支团队，换金融题(信用卡违约)零改动直接跑 → github.com/andyxueanan-dot/modelcrew」 |

## 录制提示
- **镜 5 是全片重点**，可以放慢、加个红框圈住那个 ❌，停留久一点（这是赛道八最想看的"AI 怀疑 AI"）。
- 如果完整跑一遍太久，可**预先跑好**，录屏时滚动展示已有产出（`cases/2024_mcm_c_tennis/artifacts/`），重点录"输入一句话 + Router 召唤 + Critic 判 ❌"这三段，中间快进。
- 字幕用大号字、简短；不需要专业配音，打字幕最稳。
- 导出 mp4（≤ 50MB 便于论坛上传）；也可转 gif 放进 forum_post.md。
