# 样例 · 2024 MCM Problem C —— 网球的"动量"(Momentum in Tennis)

> 这是用来**验证框架能端到端跑通**的 demo。换成别的赛题时，新建一个 `cases/<赛题>/` 即可，
> `agents/` 里的团队定义不用改——这正是框架可复用的证明。

## 题目（原题要点）
用 2023 温网男单第二轮后的逐分数据，建模量化比赛中的"动量(momentum)"：
- Q1：能否量化任一时刻"谁打得更好、好多少"？
- **Q2（关键）**：一位教练认为"连胜只是随机"——请评估"势头/连胜"到底是不是随机的。
- Q3：能否预测局势反转(swing)？哪些因素最重要？
- Q4：模型能否推广到其他比赛/其他运动？

## 为什么用它做 demo
Q2 本身就是统计学经典争议 **热手谬误 (hot-hand fallacy)**，
正好让框架的 ⭐**Critic Agent** 踩中一个真实学术争议——
"一群 AI 建出动量模型，另一个 AI 去证伪它"，是整套作品的高光。

## 本样例的运行路线（由 Router 路由为"数据型 C 题"）
Analyst → Scout(重) → Modeler(EWMA 动量 + 反转预测) → Solver → **Critic(随机性/热手谬误检验)** → Writer

各 Agent 产出存于 `artifacts/`，数据见 `data/`。
