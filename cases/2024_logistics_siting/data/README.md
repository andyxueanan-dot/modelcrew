# 数据 · 应急服务站选址

- `communities.csv` —— 64 个社区，字段 `id, x, y, population`；坐标在 0–100km 城区内。
- `generate_data.py` —— **数据生成脚本**（固定 `np.random.default_rng(42)`，逐字节可复现 `communities.csv`）。
  明确记录数据由 **5 个居住组团**生成，便于独立核对组团结构（回应 Critic"组团数应可复核"的审计意见）。

## 字段
| 字段 | 含义 |
|------|------|
| id | 社区编号 |
| x, y | 社区坐标（km，城区 0–100） |
| population | 社区人口（作为加权权重） |

## 关键事实
- 总人口 **346,573**；由 **5** 个居住组团构成（中心约 (12,12)/(12,78)/(80,15)/(82,80)/(48,46)）。
- 这 5 组团结构是本案例的"地面真相"：选 **K=4** < 5 必然要合并两个组团 → 成本随 K 在 **K=5** 处有明显拐点（见 results.json 敏感性）。
