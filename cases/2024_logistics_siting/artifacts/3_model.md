# 3 · 建模 Modeler · 连续加权 p-中位

## 模型定义
- **决策变量**：K=4 个站点的连续坐标 {c_k = (x_k, y_k)}，k=1..4。
- **目标**：min ∑_{i=1}^{64} w_i · min_k ‖p_i − c_k‖₂
  （w_i = 社区 i 人口，p_i = 社区坐标；即人口加权"到最近站点"的总欧氏距离）。
- **约束**：c_k ∈ [0,100]²（城市边界）。
- 性质：**非凸**（min_k 与欧氏范数耦合），存在大量局部最优。

## 方法选择（写清为什么 + 替代方案 + 代价）
| 方法 | 说明 | 取舍 |
|---|---|---|
| **交替局部搜索**（采用）：最近分配 ↔ 各簇 **Weiszfeld** 求 Weber 点 | Weiszfeld 是欧氏 1-中位的精确迭代解；比 k-means 重心更新更贴合"距离和"(非平方)目标 | 收敛快、可解释；但**只保证局部最优** |
| **k-means++ 多起点**（采用）：分散播种 + 多次重启取最优 | 缓解局部最优陷阱，可靠逼近全局 | 仍是启发式，**无全局最优证书**（见 Critic C4） |
| 离散 p-中位 MILP（备选未用） | 若候选站点离散且可枚举，可求**证明最优** | 连续选址不适用；规模大时不可解 |

> 关键设计（防自相矛盾）：headline 最优解**直接取敏感性曲线上 K=4 的那一点**，
> 即"主结果"与"敏感性扫描"用同一函数、同样起点数算出 → 二者天然一致，杜绝口径裂缝。

## 待审计结论清单（交给 ⭐Critic）
- **C1**：naive 单次局部搜索得到的布局就是最优解。〔置信度低，疑似局部最优〕
- **C2**：最优布局 4 站点均在城市边界内，满足可行性。〔置信度高〕
- **C3**：K=4 是最优站点数。〔置信度低，K 是题设预算而非优化结论〕
- **C4**：multistart 解即全局最优。〔置信度低，启发式无证书〕
- **C5**：模型可直接用于真实应急规划。〔置信度低，假设强〕
