# 0 · 路由结论（Router · 调度中枢）

## ① 题型分类
**优化型（离散优化/运筹混合，连续选址）**。依据：决策变量是 4 个站点的连续坐标，目标为人口加权总距离最小化的**连续加权 p-中位问题**——非凸、多局部最优。**与本仓库前两个数据型案例（网球动量、信用卡违约）不同**，正好验证"换题型即用"。

## ② 召唤专家与任务
- **Analyst**：解析目标/约束，明确"人口加权""最近站点"语义，定数据口径。
- **Modeler**：建连续 p-中位模型，选启发式（Weiszfeld + 多起点重启 / K-means 式交替）。
- **Solver**：多随机初值求解，记录最优布局与目标值。
- **Critic（硬闸门）**：可行性 + 最优性核验。
- **Writer**：成文。

## ③ 协作时序
Analyst → Modeler → Solver → Critic（闸门）↺ → Writer

## ④ Critic 硬闸门重点
① **局部最优≠全局最优**：是否足量多起点、解是否稳定收敛同一布局；② 可行性：坐标在城域内、权重非负；③ **K 的选择**：K=4 是否做了敏感性、边际收益拐点；④ 真实应急风险：用直线距离替代道路/响应时间的偏差。
