### 最优性论证（Q1-d）
本题命门：**消耗掉的资源不退款**（消耗 1 箱水净亏全额买价 5 元，仅剩余资源半价退），且**负重 ≤ 1200 kg 把"远赴矿山多挖矿"所需的大量囤货卡死**。最优性由两条独立证据互证：

1. **Range A — 精确前向 DP（cap 内）+ 挖矿证书取 max**：
   - 状态 `(t, p, w, f)`，负重显式进状态，现金维由价值函数 dominance 吸收。携带上限 cap 自小而大（40→…）升至**非绑定**（最优携带 42/38 严格 < cap=55）→ 证明 DP 在该携带域内精确。
   - 远端高携带的「挖矿路线」由**独立精确证书** `_best_mining_wealth` 覆盖：BFS 最省消耗进/出矿山 × 逐日真实天气 × 精确补给 LP（起点价 vs 村庄价、负重逐日 ≤ 1200、消耗全价、剩余半价退）。基线下**最优挖矿财富 = 8435 元 < 直达 9410** → 挖矿不划算。
   - 全局最优 = max(直达 DP 9410, 挖矿证书 8435) = **9410**。
2. **Range B — 缩规模独立后向 DP 对拍**：在同一张图、缩短 horizon（T=6 与 T=8）上，用**完全不同的编码**（后向递推 + 显式购买档位枚举，不引用 A 的解/界）独立求解，**两个规模均得 A=B=9410，差 0** → 双范式互证。
   - （全规模 B 的状态数 ≈ 上亿，纯 Python 必爆内存，故按建模原意 B 仅作缩规模独立交叉验证。）

> 直观解释：矿山（12 号）距起点 8 步、距终点 5 步，绕去挖矿至少多走 10 天；在 1200 kg 负重下要么一次背不动全程囤货（载重超限），要么进村按 2 倍价补给——挖矿 1000 元/天的收益跑不赢"绕路消耗 + 囤货净亏"。故最优是 3 天直达，**这与"早到省消耗 vs 绕矿多挖钱"的核心张力一致，由优化结果判定为前者**。

## 二、Q2 在线稳健决策（每天只知当天天气）

### 策略：沙暴安全垫 + 滚动重规划
沿 BFS 主干路径（1-25-26-27）前进；沙暴日强制停留；起点按"主干步数 × 最坏（沙暴）单日消耗 + 安全垫 buffer 天"配置资源；途经村庄按剩余路程滚动补给。`buffer` 为可调安全冗余（沙暴当量天数）。**严格只用当天 + 历史天气**（PoC-3 信息墙已验，无未来泄漏）。

### 损失评估（Q2-b，真实 30 天序列）
| buffer | 在线财富 | vs Q1(9410) 损失 |
|---|---|---|
| **0（最优）** | **9255** | **155（1.65%）** |
| 1 | 9180 | 230 |
| 2 | 9105 | 305 |
| … | … |（每加 1 天安全垫 ≈ 多囤资源半价退 ↘ 75 元）|
| 6 | 8805 | 605 |

- **最小损失 155 元（1.65%）**，即"看不到未来"相比"上帝视角"的代价。
- **信息价值（Value of Information）= Q1 − Q2 = 155 元**：这是"提前知道整段天气"对该玩家的经济价值（信息经济学视角）。

### 最大风险天气波动（Q2-c）
对抗序列压力测试（离线最优用精确 A 同口径重算）：

| 对抗序列 | 默认 buffer(0) | **最小可幸存 buffer** | 离线最优 |
|---|---|---|---|
| 早期分散沙暴（2,3,5,6 日） | **失败（断粮）** | **5** | 8510 |
| 起始连续沙暴（1,2,3 日） | **失败** | **3** | 8670 |
| 前 5 天全沙暴 | **失败** | **5** | 8510 |
| 前 7 天高温 | 存活（9250） | 0 | 9400 |

**结论：风险最大的天气波动 = 行程早期的连续/密集沙暴**——被困期间持续消耗却不能前进，低安全垫策略直接断粮失败。**稳健性—财富权衡**：buffer=0 在真实序列下财富最高，但在对抗序列下崩溃；要在"早期密集沙暴"下保证到达终点，需 buffer ≥ 5（对应损失增大）。这定量刻画了 Q2 稳健性的代价。

## 三、Q3 敏感性与局限

### 单因子敏感性 + 临界点（Q3-a / Q3-c）
所有点用全局精确引擎（DP + 挖矿证书）求解，红/蓝表示挖矿是否变为最优：

| 参数 | 行为 | **临界点** |
|---|---|---|
| **矿山收益 M** | M ≤ 1200 直达最优 9410；**M ≥ 1300 挖矿(4天)变最优** | **M\* ≈ 1300 元/天**（基线 1000 在阈下，挖矿不划算） |
| **负重上限 W** | W ≤ 1200 直达 9410；**W ≥ 1400 挖矿变可行/最优** | **W\* = 1400 kg**（题面 1200 恰在阈下，挖矿被负重卡死） |
| **截止天数 T** | T ∈ [25,35] 均 9410（直达仅需 3 天，时间窗极宽裕） | 最小可行 **T = 25**（已是扫描下界，更小未测） |
| **消耗缩放 s** | s=1.0→9410；s↑→财富线性↘（1.2→9300）；**s≤0.9 时挖矿变最优**（消耗变便宜，挖矿可持续） | s\* ≈ 0.9 附近 |
| **退回比例** | 0→1.0 倍均 9410 **不变**（最优恰买够、剩 0 无退款，故退价不影响） | 无（结构不敏感） |
| **村庄价倍率** | 倍率 ≥ 1.5 均 9410 不变；**=1.0（村价=起点价）时挖矿变可行 9515** | 倍率 ≈ 1.0 附近 |

> **关键洞察**：基线参数 (M=1000, W=1200kg) **恰落在两条挖矿临界线的"挖矿不划算"一侧且都很接近**——矿山收益再涨 30%（M≥1300）或负重再松 17%（W≥1400）就会翻转为"绕去挖矿"。这说明本题设计精巧：直达与挖矿在基线下势均力敌，是经过调参的临界平衡。退回比例与村庄价对基线最优**完全不敏感**，因为最优方案恰买够、不进村、不留余。

### 单调性体检（PoC-4，模型自洽）
- 矿山收益 M:1000→0，最优财富 9410→9410（**不增 ✓**，少了赚钱手段，单调不增）。
- 截止天数 T:30→31，最优财富 9410→9410（**不减 ✓**，约束放松，单调不减）。

### 模型局限与适用边界（Q3-b）
1. **动作离散整天三选一、整数箱**：现实中"半天移动半天挖矿"或分数箱会略改最优值。
2. **天气离线完全已知（Q1）**：Q1 是 Q2 的性能上界；现实近 Q2 在线情形。
3. **单矿山、单村庄、负重瞬时约束**：地图扩大、多矿/多村会改变挖矿—补给的几何经济。
4. **无设备故障 / 无随机事件**：确定性假设；现实有抛锚、迷路等。
5. **挖矿"到达即挖"口径**：本模型按"到矿山要先花一天移动，之后才挖矿"（动作互斥）；若出题允许"到即挖"，挖矿收益与上述阈值会下移。
6. **缩规模对拍的范围**：Range B 仅在 T≤8 上独立验证（全规模纯 Python 不可行），故"双范式互证"覆盖的是 A 的核心 DP 逻辑（直达 + 近端补给），远端挖矿由独立证书而非 B 覆盖。
