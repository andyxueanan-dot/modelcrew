# Problem Analysis — 信用卡违约预测

## Sub-Questions

| ID | Restatement | Type | Depends On | Evaluation Hint |
|----|-----------|------|-----------|----------------|
| Q1 | 能否用客户特征预测下月是否违约？构建分类模型。 | Required | — | AUC-ROC、KS statistic、F1-score（不只看准确率） |
| Q2 | 哪些因素对违约最重要？给出特征重要性排序。 | Required | Q1 | Feature importance / SHAP values；可解释性 |
| Q3 | 在 22% 违约率（类别不平衡）下，如何正确评估模型？ | Required | Q1 | AUC、PR 曲线、混淆矩阵、recall/precision 分层报告 |
| Q4 | 把模型用于真实风控，有什么合规与公平风险？ | Required (KEY) | Q1–Q3 | 敏感属性审计；公平信贷法规分析；整改建议 |

## Key Definitions

| Term | Definition | Source |
|------|-----------|--------|
| **违约 (Default)** | `default.payment.next.month = 1`：客户在下一个账单周期未能偿还最低还款额 | UCI dataset documentation |
| **信用评分** | 模型输出的违约概率 P(default=1)，范围 [0,1]。越高 = 越可能违约 | 模型输出 |
| **PAY_0..6** | 还款状态编码：-1=按时还款，-2=无消费，0=使用循环信用，1–8=逾期月数。PAY_0=最近月，PAY_6=6个月前 | UCI documentation |
| **LIMIT_BAL** | 信用卡授信额度（新台币） | UCI documentation |
| **KS statistic** | Kolmogorov-Smirnov 统计量：衡量模型区分好坏客户的能力 = max|TPR - FPR| | 信用风险建模标准指标 |
| **准确率陷阱** | 在类别不平衡数据上，accuracy 可能被多数类主导，导致对少数类（违约客户）无区分力 | 机器学习常识 |

## Assumptions

| # | Assumption | Why | Fragility |
|---|-----------|-----|-----------|
| A1 | 数据来自台湾某银行 2005 年 4–9 月的真实信用卡客户 | UCI 数据集文档 | 2005 年的台湾信用卡市场与今天差异巨大（监管、经济环境、客户行为） |
| A2 | PAY_0 反映的是"最近一个月"的还款状态，不包含未来信息 | 字段命名约定 | 如果 PAY_0 实际包含了"下月"的某些信息，就会造成数据泄漏——**这是 Critic 必须审计的核心风险** |
| A3 | 24 个特征均为已知、可用的客户信息 | 数据集提供 | SEX/AGE/EDUCATION 在真实银行场景可能受公平信贷法规限制 |
| A4 | 违约标签 (default.payment.next.month) 是准确的 | 银行系统记录 | 标签可能存在噪声（银行系统记录延迟、定义口径差异） |
| A5 | 30,000 条记录独立同分布 (i.i.d.) | 标准 ML 假设 | 同一家庭/关联账户可能违反独立性 |
| A6 | 训练集/测试集的随机划分能代表部署场景 | 标准验证方法 | 如果部署时客户分布发生漂移 (concept drift)，模型性能可能下降 |

## Traps & Warnings

1. **准确率陷阱**：违约率 22.1%，"全预测不违约" baseline = 77.9%。如果模型达到 80% 准确率，实际可能毫无价值——必须用 AUC/KS/F1 评估。

2. **PAY_0 的数据泄漏风险**：PAY_0 是"最近一期还款状态"。在真实预测场景中，我们只能在"当前时间点"预测"下月"是否违约。如果 PAY_0 编码的是"当月"的还款状态，而违约标签是"下月"——那么 PAY_0 可能已经部分反映了客户当月的财务困境，这不构成泄漏。但如果 PAY_0 包含了下月的信息，则是严重泄漏。**必须明确 PAY_0 的时间锚点。**

3. **敏感属性合规风险**：SEX(性别)、AGE(年龄)、EDUCATION(教育)在多数国家的公平信贷法规中属于受保护属性。将违约概率与这些属性挂钩可能导致歧视性拒绝贷款——违反美国的 ECOA (Equal Credit Opportunity Act) 或类似法规。

4. **类别编码异常**：EDUCATION 包含 0,4,5,6（文档未定义），MARRIAGE 包含 0（文档未定义），PAY_0..6 包含 -2（无消费）和 8（逾期 8+ 月）。异常值需要仔细处理。

5. **概念漂移**：2005 年的数据用于训练，如果部署在 2024 年的客户群上，经济环境、监管政策、客户行为均已变化。模型需要定期重新训练。
