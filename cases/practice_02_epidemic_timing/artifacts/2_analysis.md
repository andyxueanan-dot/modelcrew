# 审题分析 · 练习题 02 传染病干预时机

> 角色：Analyst（审题官）。本文件**仅依据 README.md** 从零审题，未参考任何 `_contaminated_orchestrator_authored/` 归档内容。
> 题型（题目自报，仅供归档）：机理/连续型。Analyst 复核同意：这是一道**确定性常微分方程（ODE）室室模型 + 政策参数优化 + 敏感性/稳健性分析**题。

---

## 0. 一句话题意

给定一个 R₀=3.0 的人传人传染病和明确人口/恢复率参数，要求：(Q1) 建立机理模型并刻画无干预自然流行；(Q2) 在第 τ 天起把传播率降低比例 c，找到能让"同时在治"峰值不超过医疗承载 h=8%N 的干预时机，并给出对参数不确定性稳健的建议；(Q3) 评估结论对关键参数的敏感性与模型适用边界。

---

## 1. 子问题清单（必答 / 加分）

| 编号 | 子问题 | 性质 | 交付物 |
|---|---|---|---|
| **Q1-a** | 建立刻画人传人传播的数学模型（写出方程、状态变量、参数含义） | **必答** | 模型方程 + 参数标定（由 R₀、γ 反推 β） |
| **Q1-b** | 在无干预下给出**基本再生数 R₀**（题目已给 3.0，需用模型自洽地推导/验证，而非照抄） | **必答** | R₀ = β/γ 推导 + 数值核对 |
| **Q1-c** | 无干预下**同时在治感染者峰值规模**（I 的最大值，绝对人数与占 N 比例）及其**出现时点**（峰值天数 t_peak） | **必答** | 峰值 I_max、t_peak |
| **Q1-d** | 无干预下**最终累计感染比例**（流行结束时被感染过的人口占比，即 attack rate / 1−S(∞)/N） | **必答** | 最终规模 z（数值解或最终规模方程） |
| **Q2-a** | 引入干预："第 τ 天起把 β 降低比例 c 并一直维持"。先**合理选定一个干预力度 c**（README 明确允许"合理选定"） | **必答** | 选定的 c 值 + 选取理由 |
| **Q2-b** | 在该 c 下，求**能使峰值同时在治 ≤ h 的干预时机 τ**（关键：是求 τ 的**可行区间/临界 τ\***，越晚越难达标） | **必答** | 临界 τ\*（最迟可行干预日）/ 可行 (τ,c) 区域 |
| **Q2-c** | 针对**关键参数不确定**（R₀、γ、c 执行力度、h 等）给出**稳健的**时机/力度建议（不是单点最优，而是在参数波动下仍达标的方案） | **必答** | 稳健 τ/c 建议 + 稳健性论证（如最坏情形、安全裕度） |
| **Q3-a** | 结论对关键参数（R₀、c、h 等）的**敏感性**评估 | **必答** | 敏感性分析（单参数扫描 / 局部偏导 / Sobol 等择一） |
| **Q3-b** | 模型的**局限与适用边界**（如实评估） | **必答** | 假设失效场景清单 + 适用范围声明 |
| **加分** | 可行域可视化（τ–c 等高线/相图）、峰值随 (τ,c) 的曲面 | 加分项 | 图 |
| **加分** | 多波/季节性、报告延迟、干预成本权衡（"最小 c 使最迟 τ 可行"）、最终规模与峰值的权衡 | 加分项 | 拓展讨论 |

> **拆题要点**：Q2 的核心不是"随便找一个能达标的 τ"，而是题面"求能使峰值不超过 h 的干预**时机**"——这天然指向**最迟可行干预日 τ\***（given c）或**可行 (τ,c) 区域的边界**。Modeler 不要把它降级成单点问题。

## 2. 关键定义表

| 名词 | 可操作定义 | 来源 |
|---|---|---|
| 传染病模型 | 采用 **SIR**（易感 S / 感染在治 I / 移除 R）确定性 ODE 室室模型作为基线。README 只说"人际接触传播""平均传染期""基本再生数""恢复率 γ"——这些恰好是 SIR 的标准参数集，无暴露期/无症状/年龄结构信息 → SIR 为最小自洽模型 | **Analyst 假设**（贴合题面参数集；SEIR 为加分对照） |
| N | 总人口 = 1,000,000，闭合人群（无出生/死亡/迁入迁出） | 题目原意 |
| I₀ | 初始感染在治者 = 10；S₀ = N−I₀ = 999,990；R₀(初值)=0 | 题目原意（注意符号冲突，见陷阱 T1） |
| γ（恢复/移除率）| ≈ 1/6 /天（平均传染期 6 天 ⇒ γ=1/6） | 题目原意 |
| R₀（基本再生数）| = β/γ = 3.0；据此反推 β = R₀·γ = 3·(1/6) = **0.5 /天** | 题目给 R₀；β 由 Analyst 反推 |
| β（传播率/有效接触率）| 单位时间每名感染者产生的有效接触致感染率，标定为 0.5/天（无干预） | Analyst 反推 |
| "同时在治"感染者 | 模型中的 **I(t)**（当前正在传染期/在治的人数），**非**累计感染。容量约束作用于 I(t) 的峰值 | 题目原意（"同时在治"=prevalence，非 cumulative） |
| h（医疗承载上限）| = 8% × N = 80,000 人。判据：max_t I(t) ≤ h 即不挤兑 | 题目原意 |
| 干预力度 c | β 的**降低比例**：干预后 β' = β·(1−c)，c∈(0,1)。c=0 无干预，c=1 完全阻断 | 题目原意（README 明确"降低比例 c"） |
| 干预时机 τ | 干预起始日（天）。t<τ 用 β；t≥τ 用 β(1−c)，并**一直维持** | 题目原意 |
| 干预后有效再生数 R_e | = β(1−c)/γ = R₀(1−c)（瞬时，未计 S 耗损）；> 1 时干预后仍会继续增长 | Analyst 推导 |
| 最终累计感染比例 z | 1 − S(∞)/N，流行终止时被感染过的人口占比；满足最终规模方程（无干预下） | 题目原意 + 标准定义 |
| 稳健的建议 | 在关键参数落入合理不确定区间（最坏情形）下，方案仍满足 max I ≤ h，并留安全裕度 | Analyst 操作化（README 只说"稳健"） |

## 3. 建模假设清单（每条附"若不成立的代价"）

| # | 假设 | 为什么这么假设 | 若不成立的代价 |
|---|---|---|---|
| A1 | 采用 **SIR**（无潜伏期 E） | 题面只给传染期与 R₀、γ，无潜伏/无症状信息；SIR 是与给定参数自洽的最小模型 | 若实际有较长潜伏期（SEIR），峰值会**更晚更平**，且干预"见效有延迟"——同样 τ 下 SIR 会**高估**所需的提前量，可能让 τ\* 估计偏保守或偏激进。建议 Q3 用 SEIR 做对照 |
| A2 | 人群**均匀混合**（mass-action，βSI/N） | 标准室室模型假设；无接触网络/空间数据 | 真实接触异质（超级传播、网络结构）会使 R₀ 等效值偏移、峰值更尖；均匀混合通常**低估**早期增速 |
| A3 | 人口**闭合**、无人口学出生死亡、无再感染（终身免疫） | 流行时间尺度（数月）远短于人口更替；R 终身免疫是 SIR 默认 | 若有免疫衰减/再感染，会出现**多波**，单峰分析失效；最终规模被低估 |
| A4 | 参数**时不变**（β、γ 常数），干预为**阶跃**（τ 时刻 β 瞬间降到 β(1−c) 并恒定） | 题面字面："第 τ 天起降低 c 并一直维持" | 真实干预有**爬坡/疲劳/反弹**；阶跃假设会**高估**干预的即时与持久效果，使 τ\* 偏乐观（实际需更早干预） |
| A5 | c 是**确定且可完美执行**的 β 折减 | 题面把 c 当政策变量 | 执行力度有偏差（依从性<100%），等效 c 偏小 → 实际峰值更高。这正是 Q2 稳健性要覆盖的不确定源 |
| A6 | γ=1/6 取"约 6 天"为精确值；β=R₀γ=0.5 由 R₀=3 反推 | 题目给的是"约"和"≈"，需取定值才能算 | R₀、γ 本身有不确定（"约""≈"），Q3 必须扫描；R₀ 偏高一点 → 峰值显著上移、τ\* 显著提前（高度敏感，见 T4） |
| A7 | 容量约束作用于**瞬时 I(t)** 的峰值，h 为硬阈值 | "同时在治""承载上限""超过即挤兑" | 若 h 实为住院/重症子集（仅部分 I 需床位），则有效阈值更宽松，本模型偏保守。需向出题人核实 I 与"在治"是否等同（标**需核实**） |
| A8 | 数值上确定性求解（固定求解器/步长/容差），无随机性 | README 要求字节级可复现 | 步长过大或刚性处理不当会让峰值/t_peak 数值漂移；需固定 solver 设置 |

## 4. 已知陷阱 / 易误解清单（交 Modeler 与 Critic 重点盯）

- **T1 符号冲突（高危）**：题面里 "R₀" 同时被用作**基本再生数 R₀=3.0**和**初始移除者 R(0)**两种含义。务必区分：基本再生数记 **𝓡₀ 或 R0_repro**，移除室初值 R(0)=0。报告与代码命名必须消歧，否则全队读串。

- **T2 "同时在治" ≠ "累计感染"（高危）**：h 约束的是 **prevalence I(t) 的峰值**，不是 attack rate / cumulative。Q1-c 求 I_max，Q1-d 求最终累计 z=1−S(∞)/N，两者是不同量，别混。挤兑判据是 max I(t) ≤ 80,000。

- **T3 干预后仍可能增长**：当 R₀(1−c) > 1（即 c < 1−1/R₀ = 1−1/3 ≈ 0.667）时，干预**不能立刻让 I 下降**，只是减缓；峰值仍可能出现在 τ 之后。因此"选定合理 c"时要明确：若想靠干预直接压住，c 需足够大；若 c 适中，则**τ 必须足够早**才能让峰值卡在 h 以下。Q2 是 τ 与 c 的**耦合**问题，不能只调一个。

- **T4 临界时机 τ\* 的方向性与高敏感性**：干预越晚，S 已被消耗、I 已积累，越难把峰值压到 h 以下 → 存在**最迟可行 τ\***（given c）。且 τ\* 对 R₀ 极敏感：早期 I 近似指数增长 e^{(β−γ)t}，R₀ 小幅上调会让"达标窗口"显著左移。这正是为什么 Q2 必须给**稳健（留裕度/按最坏 R₀）**的建议，而非单点 τ\*。

- **T5 "峰值不超过 h" 的边界判定**：注意无干预峰值远超 h（粗估 SIR 在 R₀=3 时 I_peak≈0.3N=300,000 ≫ 80,000），所以**不干预必挤兑**，Q2 一定有非平凡解空间。但若 τ 太晚或 c 太小，可行解可能不存在——要诚实报告"在给定 c 下无可行 τ"的情形，而不是硬凑。

- **T6 初值与单位一致性**：S₀=N−I₀ 而非 N；时间单位统一为"天"；β、γ 单位 /天。比例量（h、c、z）与绝对量（I_max、N）换算别错（h=0.08×N=80,000）。

- **T7 "约/≈" 即不确定性信号**：题面用"约 6 天""R₀≈"暗示参数本身不精确，Q2 的稳健性与 Q3 的敏感性正是对此回应——不要把给定值当无误差真值后才在 Q3 补敏感性，应在 Q2 就把不确定性纳入。

- **T8 加分项别喧宾夺主**：SEIR、成本权衡、多波等是加分对照，但**必答的 Q1 数值（I_max/t_peak/z）、Q2 的 τ\* 与稳健建议、Q3 的敏感性**必须先齐全且可复现。

## 5. 给下游的交接备注

- **Modeler**：基线用 SIR（A1），β=0.5、γ=1/6、N=1e6、I₀=10。Q1 出 I_max、t_peak、z 三个数；Q2 把 (τ,c) 当二维政策空间，先固定一个合理 c（建议在 T3 的临界 c≈0.667 附近或其上下各取，便于讨论），扫 τ 求临界 τ\*，再画可行域；Q2 稳健性按"R₀ 取区间上端 / c 打折扣"求最坏情形 τ。固定 solver 设置保证可复现（A8）。
- **Critic**：重点查 T1（符号串用）、T2（prevalence vs cumulative）、T3（c<0.667 时干预后仍增长是否被正确建模）、T5（可行解存在性是否被诚实报告）。
- **需核实（不替出题人脑补）**：T7-A7 中"同时在治"是否等同于"全部 I"还是仅需床位的子集——README 未明确，按"全部 I"取保守解，并在报告标注该口径。
