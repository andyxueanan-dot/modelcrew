# 练习题 02 · 传染病干预时机（机理型 / 连续动力系统）

> ModelCrew 自练案例（`/loop` 刷题 iter2）。题型：**机理型（连续/ODE）**，方法箱：微分方程动力系统。
> 选它因为现有案例（体育统计 / 金融 ML / 选址优化 / 因果混杂 / 时序预测）**全是离散/数据型，没有一个用过连续动力系统方法箱**——最能检验框架"换题型即用"，且干预时机问题天然有"阈值 + 决策"结构，给创新落点（命名 + 稳健性指标）留足空间。

## 题面

某地暴发一种通过接触传播的传染病，可用 SIR 仓室模型刻画（总人口 N、初始感染 I₀）。卫生系统对**同时在治感染者**的承载上限为 H（占总人口比例 h）。当局可在第 τ 天起实施干预（社交限制），把有效接触率降低比例 c（如戴口罩/限聚使传播率降为 (1−c)β），并一直维持。

- **Q1**：建立 SIR 模型，刻画**无干预**下的疫情：基本再生数 R₀、峰值同时感染人数与峰值时点、最终累计感染比例（最终规模）。
- **Q2**：引入"第 τ 天起降低接触率 c"的干预，求**使峰值同时感染不超过容量 H 的最晚干预起始日 τ_max**（干预时机窗口）。并就参数不确定性给出**稳健的**起始日建议。
- **Q3**：结论对关键参数（R₀、c、H）有多稳健？如何诚实评估模型局限？

## 实质

不是"解一个 ODE 画条曲线"，而是：**机理建模 → 求阈值结构（R₀、峰值条件 S/N=1/R₀）→ 把"何时干预"化为带容量约束的决策窗口 → 用一个可报告的稳健性指标量化参数不确定性对决策的影响**。考点正是机理题的得奖要素：阈值分析 + 决策导向 + 诚实的稳健性量化。

数据/参数确定性给定，`artifacts/solve.py` 数值积分（`scipy.solve_ivp`），字节级可复现。

## 创新落点（建模阶段即埋，回应 innovation_boost 复盘）

- **方法命名**：「**阈值–窗口法**」——把阈值动力学（R₀、峰值条件）与容量约束下的干预时机窗口统一成一套可复用流程。
- **可报告稳健性指标**：「**干预窗口稳健半径 IWRR**」= 名义最晚安全起始日 τ_max 与传播率上浮 20%（坏情形）下最晚安全起始日之差（天）。它把抽象的"敏感性分析"操作化成一句决策语言：**"参数不确定性会吃掉你 ρ 天的提前量——应比名义截止日提前 ρ 天行动。"**
