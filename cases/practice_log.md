## 🔒 纯净基准协议（iter2 用户纠偏后确立，刷题必须遵守）

基准要有效，**解题与评分都必须 clean-context**，否则量的是"带全上下文的编排者"而非"框架本身"：

1. **解题无菌**：建模/求解/写作交给**新开的 modelcrew 角色子代理**（analyst→modeler→solver→writer），各自 fresh context。编排者**只在角色间传递工件**，**绝不**亲手写 solve.py / 论文，**绝不**注入解法、数字、命名、或**从先前跑动学到的过程经验**（连"图要加中文字体回退""别用 R₀ 下标"这种踩坑 tip 都算泄漏——iter2 实证）。
2. **题面无菌**：README 只写情景 + 给定参数 + 问题，**不得**预埋方法/创新点（旧 iter2 README 曾把"阈值–窗口法+IWRR"写进题面=污染，已剔除）。
3. **评分无菌**：每次打分**全新 Judge + 中立 prompt**，不透露目标分/历史分/作者意图；**禁止续跑同一评委"重评提分"**（实测虚高 89 vs 盲评 86）。评委须 Read 打开 figures/*.png 真看图。
4. **污染隔离**：被污染的旧解归档到 `artifacts/_contaminated_*`（`_` 前缀=各角色跳过），并在 prompt 明令禁开。
5. **编排者本身有上下文**这一点不可完全消除——最彻底是连路由也交给无上下文 router；当前以"只传工件、不注入内容"为底线，并在本表如实标注任何残留泄漏。

> 当前状态：demo_dispatch=纯净；iter1/iter2(旧)=编排者亲写**已作废**；iter2′(84)、**iter3 穿越沙漠真题(83)=纯净**。
> **iter3 残留泄漏如实标注（协议第5条）**：编排者为忠实转写真题地图，从 GitHub 权威仓库取数据时**同时看到了一份参考解**（答案在编排者脑中）。缓解=全程只给各 agent 文件路径、**绝未把答案/方法/数字写进任何 prompt**；且 **Critic 独立穷举复算**(不知编排者所想)与 **Judge 盲评**两道闸门可证答案未泄漏进 agent——实证：Critic 反而独立推翻了编排者"9410 是假最优"的私下怀疑。
