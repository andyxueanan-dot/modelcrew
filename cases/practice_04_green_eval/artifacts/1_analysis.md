# 1 · 审题分析（Analyst 工件）

> 本文件只搭建子问题骨架、精确定义、假设清单、评价管线的待定环节与已知陷阱。
> **不选具体方法**（不预设熵权/CRITIC/PCA/TOPSIS/灰色关联…）、**不给数值答案**、**不预设排名**。
> 题面与数据见 `README.md` 与 `data/indicators.csv`；题型路由见 `0_routing.md`。

---

## 〇、一句话理解

把 10 个地区（A–J）在 7 项方向不一、量纲各异的指标上的截面统计值，经「方向统一 → 无量纲化 → 赋权 → 聚合」压成一个可解释的综合得分并排序，并对结论做短板诊断与稳健性评估。**唯一输入**是 `data/indicators.csv`，要求**客观、可解释、确定性、字节级可复现**。

## 一、子问题清单（含必答 / 加分标注）

## 三、综合评价问题的形式化骨架（管线 + 待定环节）

> 标准 MADM 管线分四段。**每段都有多解**，此处**只列待定环节与判断维度，不锁定方法**（方法选择交 Modeler 在 CP2 给 ≥2 套方案）。

设决策矩阵 `X ∈ R^{10×7}`，行=地区 i∈{A..J}，列=指标 j∈{1..7}；指标方向向量已知（3 逆 4 正）。

### 阶段 0 · 数据体检（交 Scout 轻量执行，非建模决策）
- 核对方向是否与 README 一致；查异常值/极值；查指标间相关性（高相关会影响某些客观赋权的"信息量"判断）。

### 阶段 1 · 方向统一（待定环节 ①）
- 逆向指标转正向。**待定**：用"取负/倒数/极差反转/max−x"等何种转向方式——不同方式与后续无量纲化耦合，影响数值。
- ⚠ 必须保证转向后**所有指标同向（越大越好）**，否则全盘错（Critic 头号审计点）。

### 阶段 2 · 无量纲化 / 标准化（待定环节 ②）
- 消除单位与量级差异。**待定**：极差归一(min-max) / z-score / 向量归一 / 均值化 等，**Analyst 不选**。
- ⚠ 口径风险：
  - min-max 对**极值/离群点敏感**，且会把每列拉到 [0,1] 后丢失原始离散度信息；
  - z-score 受分布形状影响、可能出负值（与某些聚合法不兼容）；
  - 向量归一保留相对比例。
- ⚠ 方向统一与无量纲化的**先后顺序与组合**需自洽（例如先 min-max 再取 1−x，与先取负再 min-max，结果可能不同）——须明确口径，不可两处各自处理导致逻辑泄露。

### 阶段 3 · 赋权（待定环节 ③，本题"客观"是硬约束）
- 确定 7 项指标权重 `w ∈ R^7, Σw=1`。**待定**：客观赋权可走多条路径（熵权/CRITIC/主成分/标准差…），是否需主客观组合——**全部不预设**，交 Modeler 给 ≥2 套供横向比较。
- ⚠ 客观赋权可能被**离散度大或与众不同的指标主导**，未必符合"绿色低碳"的领域重要性直觉；这正是 Q3 要敏感性对照的原因。

### 阶段 4 · 聚合与排序（待定环节 ④）
- 把无量纲值按权重聚合成综合得分并降序排名。**待定**：加权和 / 加权积 / 与理想解的距离型（如 TOPSIS 族）/ 灰色关联 等——**不预设**。
- ⚠ 不同聚合范式对"短板拉低总分"的惩罚程度不同（线性加权可被高分项补偿，距离型/乘积型对短板更敏感），会改变名次，须在 Q3 体现。

### 阶段 5 · 诊断与稳健性（服务 Q2 / Q3）
- Q2：按指标对综合得分缺口的贡献做归因，锁定短板维度。
- Q3：扰动权重方案 / 扰动无量纲化口径 / 扰动数据，比较排名一致性（可用排名相关系数等量化），找出最不稳的名次区段。

| A5 | 数据真实、无缺失、无录入错误，量纲与 README 单位一致 | CSV 10×7 满格无缺失，路由已登记"无缺失" | 若个别值是异常/错单位，会经无量纲化放大影响排名——Scout 体检阶段须做合理性/极值核查 |
| A6 | 综合得分主要用于**序数比较（排名）**，基数差距解释需谨慎 | 无量纲化后得分无外部物理量纲，绝对值依赖方法 | 若把得分差当成"领先多少倍/百分比"会误导决策——Writer/Solver 表述须克制，避免过度解读分差 |

| A8 | n=10 不做统计推断（无置信区间/显著性/训练-测试） | 样本极小，无总体抽样框架，是全样本截面评价非预测 | 不能用统计显著性背书排名；稳健性只能靠扰动/多方案对照（敏感性分析）来论证——已并入 Q3 |

## 五、已知陷阱清单（交 Modeler 与 Critic 重点盯防）

1. **逆向指标转向搞反或漏转**（头号低级失分点）：3 个逆向 `energy_intensity` / `carbon_intensity` / `pm25` 必须全部转成"越大越好"，且与 4 个正向同向。任一漏转/转反，整张排名报废。Critic 须逐列核对方向。
2. **无量纲化口径泄露/不自洽**：方向统一与标准化的先后、组合若两处各自为政，会逻辑打架。须用**统一可复现的口径**，明确写清先后顺序。
3. **赋权主观性与"客观"约束冲突**：题面要"客观"。若用人为拍的权重或对结果反向调参凑名次，违题且不可复现。客观赋权须由数据导出、过程透明。
4. **客观赋权被极端指标/离散度主导**：熵权类、标准差类赋权会把权重堆给"区分度大"的指标，未必等于"对绿色低碳更重要"。须警惕并在 Q3 对照。
5. **排名稳健性不足（Q3 核心）**：换一套赋权方案或聚合范式，名次可能大翻动，尤其**得分相近的中段地区**。须量化（排名相关系数等）并点名最不稳地区，不能只给一套排名就收尾。
6. **小样本 n=10 的统计误用**：不可上置信区间/显著性检验/训练-测试划分；稳健性靠敏感性分析而非统计推断（呼应 A8）。
7. **指标相关性导致重复计权**：若 `carbon_intensity` 与 `energy_intensity`、或某些环境指标高度相关，未去冗余会变相加倍其影响。Scout 先查相关矩阵。
8. **"靠后"门槛与"短板"归因含糊（Q2）**：必须**先**给出"靠后"的客观判定门槛、**再**用对得分缺口的贡献做短板归因；建议须**逐维度对应到具体指标**，禁止空泛环保口号。
9. **得分基数过度解读**：综合得分绝对值无物理意义，勿把分差解释成倍率/百分比领先（呼应 A6）。
10. **可复现性破坏**：脚本若依赖随机性、未固定列顺序/浮点处理不一致，会破坏"字节级可复现"。须确定性实现。
11. **极值/单调假设的边界**（呼应 A3）：对森林覆盖率、人均公园绿地等，极端高值未必线性更优；若发现某地区靠单一极值霸榜，须在局限中提示单调假设的代价。
