## 一、题型分类

### 结论
**综合评价 / 多属性决策型（MADM, Multi-Attribute Decision Making）**，对象固定为 10 个决策单元、7 项指标。
- 偏 **数据型（国赛 C / ICM 数据评价类）**，但样本极小（10×7），不属于"大数据/机器学习预测"类。
- 驱动性质：**数据驱动为主、机理为辅**。无外部数据、无标签、无需预测，本质是"对给定截面数据做客观赋权 + 排序"。

### 分类依据
1. **任务是排序而非预测**：Q1 要求对 A–J 给出综合得分与名次，属于无监督的多属性聚合，不是回归/分类，不需要训练集与泛化评估。
2. **指标方向明确、量纲各异**：7 项指标含 3 个逆向（`energy_intensity`、`carbon_intensity`、`pm25`）、4 个正向，单位互不相同 → 核心环节是**方向统一 + 无量纲化 + 赋权聚合**，这是 MADM 的标准管线。
3. **样本量极小（n=10）**：统计/ML 的"过拟合、随机性、训练-测试划分"几乎不适用；Critic 的审计重心应从"过拟合"转向 **赋权方法的主观性、量纲化口径、排名稳健性**。
4. **强调可复现与客观性**：题面明确要求"客观、可解释、可复现、确定性（无随机种子依赖）" → 决定了求解必须是确定性脚本，且评价方法需可被独立验证。
5. **Q3 直接点名稳健性/敏感性**：换权重方案排名是否翻动、哪些地区名次最不稳 → 把**敏感性分析**从"加分项"提升为"必答项"，Critic 与 Solver 须深度参与。

### 多标签 / 存疑说明
- 可多标签为「评价型 + 决策支持型」；ICM 维度上接近 **D（运筹/决策）** 的评价子类。
- **存疑点（交给 Modeler 在 CP2 给方案，不在此预设）**：客观赋权可走熵权法 / CRITIC / 主成分等多条路径，主客观组合赋权是否需要，排序聚合可走 TOPSIS / 加权和 / 灰色关联 / 等多条路径。Router 不指定，**要求 Modeler 给出 ≥2 套候选方案供 Critic 横向比较**（呼应铁律："题型拿不准/方法多解就召唤多套方案让 Critic 比较"）。

## 二、召唤的专家与理由

按"数据型"主线召唤，并依据本题特点（小样本、无预测、强调稳健性）做裁剪：

| 顺序 | 子代理 | 是否召唤 | 理由 / 本题特化任务 |
|---|---|---|---|
| 1 | `modelcrew-analyst` | 召唤 | 审题：拆解 3 个子问题；登记 7 指标方向与量纲差异；明确"无外部数据、确定性、可复现"约束；界定评价对象与目标 |
| 2 | `modelcrew-scout` | **轻量召唤** | 数据型常配"重 Scout"，但本题数据是唯一给定输入（10×7、无缺失、无外部源），故降级为**轻量数据体检**：量纲/异常值/指标相关性/方向核对，不做特征工程与外部数据采集 |
| 3 | `modelcrew-modeler` | 召唤（核心） | 建立评价模型：方向统一 + 无量纲化 + **≥2 套赋权方案** + 排序聚合方法。须把方法可解释性、客观性讲清，为 Q3 敏感性预留接口 |
| 4 | `modelcrew-solver` | 召唤 | 确定性脚本实现：算综合得分与排名；产出 Q2 短板维度诊断的数值依据；产出 Q3 的多权重方案对比与扰动实验（排名相关系数等） |
| 5 | `modelcrew-critic` | **硬闸门** | 审计方向是否搞反、量纲化是否泄露口径、赋权是否自洽、排名是否对权重过敏感、结论是否可复现；横向比较 Modeler 的多套方案 |
| 6 | `modelcrew-writer` | 召唤 | 出 LaTeX 投稿件：方法链路 + 排名表 + 短板诊断 + 稳健性章节 + 局限 |
| 7 | `modelcrew-judge` | 终评 | 按 rubrics 给全篇打分、预估奖级、给"只改一处"建议 |

**裁剪说明**：相对标准数据型管线，本题把 Scout 由"重"降为"轻"（无外部数据/无特征工程），把 Critic 的审计重心由"随机性/过拟合"改为"赋权主观性 + 排名稳健性 + 可复现"，以匹配小样本无监督评价的实际风险面。

## 三、协作时序（含 4 个人把舵卡点 + Judge 终评）

```
Analyst（审题：3问拆解/指标方向/约束）
   └─[🛑CP1 审题确认]── 人拍板：题型、目标、约束、给定数据边界是否理解正确
        │
Scout（轻量数据体检：量纲/异常/相关性/方向核对）
        │
Modeler（评价模型：方向统一+无量纲化+≥2套赋权+排序聚合）
   └─[🛑CP2 选模型]── 人拍板：选定主赋权/排序方案 + 保留几套作敏感性对照
        │
Solver（确定性脚本：综合得分/排名 + Q2短板诊断 + Q3多方案对比与扰动实验）
        │
Critic（硬闸门：方向/量纲/赋权自洽/排名稳健/可复现 + 横比多套方案）
        │
   ├── 通过 ──────────────────────────────────────────┐
   └── 审出❌ ─[🛑CP3 回退决策]── 人拍板：回退到 Modeler 或 Solver 重做（最多 2 轮）
                                                        │
Writer（LaTeX 投稿件：方法/排名表/短板诊断/稳健性/局限）
        │
Judge（按 rubrics 打分 + 预估奖级 + "只改一处"建议）
   └─[🛑CP4 评委复盘]── 人拍板：是否采纳 Judge 建议、是否定稿
        │
      定稿
```

### 卡点协议（默认全开，详见 `references/human_checkpoints.md`）
- **🛑CP1 审题后**：确认题型判定、3 个子问题理解、指标方向、"无外部数据/确定性/可复现"约束无误，再放行建模。
- **🛑CP2 选模型后**：人来拍板选定主赋权方案与排序方法，并确认保留哪几套候选用于 Q3 敏感性对照（避免 Router/Modeler 单方面定方法）。
- **🛑CP3 仅当 Critic 审出❌时触发**：由人决定回退到 Modeler（换方法）还是 Solver（修实现），最多 2 轮；若 Critic 全绿则跳过此卡点。
- **🛑CP4 Judge 打分后**：人复盘评分与"只改一处"建议，决定采纳与否、是否定稿。

## 四、硬闸门与回退规则

- **Critic 是硬闸门**：Modeler/Solver 的任何结论（排名、短板诊断、稳健性结论）必须过 Critic 审计，被证伪即回退重做，最多 2 轮；2 轮内未收敛则在 CP3 升级给人决策。
- **Critic ≠ Judge**：Critic 管"对不对"（方向是否搞反、量纲化是否合理、排名是否站得住、是否可复现）；Judge 管"能拿多少分"（rubrics 五视角 Panel）。
- 本题 Critic 重点审计清单（供 Critic 自行展开，Router 不预设结论）：
  1. 3 个逆向指标是否正确转向，正负向是否一致；
  2. 无量纲化是否引入口径偏差、是否对极值敏感；
  3. 客观赋权是否自洽、权重是否被极端指标主导；
  4. **排名稳健性**：换赋权方案 / 扰动数据后名次是否大幅翻动，哪些地区最不稳（直接支撑 Q3）；
  5. **可复现性**：脚本是否确定性、无随机种子依赖、字节级可复现。

## 六、给下一棒（Analyst）的交接

- **关键结论**：本题是小样本（10 地区 ×7 指标）无监督综合评价，唯一输入为 `data/indicators.csv`，要求客观赋权 + 确定性可复现，并必答稳健性（Q3）。
- **明确任务**：
  1. 拆解 Q1/Q2/Q3 三个子问题的可交付物；
  2. 逐列登记 7 项指标的含义、单位、方向（3 逆向：`energy_intensity`/`carbon_intensity`/`pm25`；4 正向：其余）；
  3. 明确约束：无外部数据、确定性生成、字节级可复现、方法须可解释；
  4. 不要选定具体赋权/排序方法（留给 Modeler 在 CP2 给多套方案）。
- **下一卡点**：Analyst 产出后停在 **🛑CP1** 等用户确认审题。
