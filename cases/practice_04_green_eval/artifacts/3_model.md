## 〇、一句话方案

把 MADM 管线拆成**四段**（方向统一 → 无量纲化 → 赋权 → 聚合排序），每段给 ≥2 个候选并横向取舍；
**主方案 M1 = 极差归一 + CRITIC 赋权 + 加权 TOPSIS**（显式处理 D1 相关性、抗短板补偿），
**对照方案 M2 = 极差归一 + 熵权法 + 加权和**（离散度驱动、线性补偿，作"不去冗余"的反面镜子），
**第三方案 M3 = PCA 降维去冗余**（独立验证 D1 的去冗余结论），**M0 = 等权**作零信息基线。
四套全部喂给 Q3 做敏感性对照（排名相关系数量化翻动），正面回应 Scout 的 D1/D2。

## 一、四段管线的候选与横向取舍

> 评价指标体系（沿用 Scout 的转向后语义）：决策矩阵 `X ∈ R^{10×7}`，行=地区 i∈{A..J}，列=指标 j∈{1..7}。
> 方向：逆向 3 项 `energy_intensity`/`carbon_intensity`/`pm25`；正向 4 项。

### 阶段 1 · 方向统一（逆向转正向）

| 候选 | 做法 | 优 | 劣 / 风险 |
|---|---|---|---|
| **S1-a 极差反转**（选用） | 在 min-max 内对逆向列用 `(max−x)/(max−min)`，正向列 `(x−min)/(max−min)` | 与 min-max 一次成型、无量纲化与转向**同一口径不打架**（治 Analyst 陷阱#2）；结果落 [0,1] | 与无量纲化耦合，换无量纲化方式须同步改 |
| S1-b 取负 `−x` | 逆向列乘 −1 后再标准化 | 简单；保留间距 | 单独取负后若接 min-max 与 S1-a 等价、接 z-score 才有别；易与无量纲化两处各自为政→泄露 |
| S1-c 倒数 `1/x` | 逆向取倒数 | 对"越小越好且趋零更优"语义贴合 | **非线性扭曲间距**，对本题强度类指标（0.35–1.55）会放大小值差距，引入额外假设，不可取 |

**立场**：方向统一**并入无量纲化同一步**完成（S1-a 主、对 z-score 方案用 S1-b），杜绝"两处各自处理"的逻辑泄露（Critic 头号审计点 D4）。

### 阶段 2 · 无量纲化

| 候选 | 做法 | 优 | 劣 / 风险 | 用于 |
|---|---|---|---|---|
| **N1 极差归一 min-max**（主） | 每列线性拉到 [0,1]，逆向用 S1-a | 量纲全消、TOPSIS/加权和都兼容、可解释（0=最差,1=最优）；CRITIC/熵权标准做法 | **对极值敏感**（D3：G 的 renewable、H 的 park、I 的 pm25 会被钉到 0/1）；丢失原始离散度绝对信息 | M1/M2/M3/M0 |
| N2 z-score 标准化（对照） | 每列减均值除 std | 保留分布形状、不被单点极值钉死 | **会出负值**（与加权积/部分 TOPSIS 不兼容）；零均值化后"最差≠0"解释变难 | Q3 口径敏感性对照 |
| N3 向量归一（对照） | 每列除以 `√Σx²` | 保留相对比例、是经典 TOPSIS 原生口径 | 0 不对应"最差"，可解释性弱于 min-max | Q3 口径敏感性对照 |

**立场**：主口径 N1（可解释性最强、与赋权/聚合都兼容）。**因 D3 极值敏感是真风险，Q3 必须用 N2/N3 复跑一遍**看排名是否翻动——这正是 Analyst 陷阱#2/#11 的兑现点，不是可选项。

### 阶段 3 · 赋权（本题"客观"是硬约束 — 这是胜负手，直面 D1/D2/A7）

> **核心张力**：Scout 的 D1（强相关簇，energy~carbon r=0.996）会让**不去冗余的赋权重复计权**；D2（CV 3.4×）会让**离散度驱动的赋权偏向高 CV 指标**。A7 又要"客观优先于领域先验"。

| 候选 | 赋权机理 | 处理 D1? | 处理 D2? | 优 | 劣 / 风险 |
|---|---|---|---|---|---|
| **W1 CRITIC**（主） | 权重 ∝ 标准差 ×(冲突性 Σ(1−r))，**对比强度 × 冲突性** | **是**（冲突性=1−相关，簇内高相关→冲突低→自动压权，治 D1） | 部分（含标准差项，但被冲突性中和） | **唯一同时显式处理相关性与离散度的客观法**；正面回应 Scout 头号埋点 D1 | 权重公式对相关矩阵估计敏感（n=10 小样本，r 估计有噪声）；可解释性中等 |
| W2 熵权法（对照） | 权重 ∝ 信息熵冗余度（≈离散度） | **否**（不看相关，D1 簇被重复计权） | **是**（强离散度驱动，D2 兑现：抬 renewable/forest、压 solid_waste） | 纯客观、经典、可复现 | **重复计权 D1 簇 + 被 D2 主导** → 正是 Analyst 陷阱#4/#7 的"反面教材"，留作对照镜子 |
| W3 PCA 主成分（去冗余对照） | 用主成分载荷构造权重 / 直接用主成分得分 | **是**（线性去相关，簇内冗余被合并进同一主成分） | — | 数学上彻底去相关、独立验证 D1 结论 | 主成分**符号/解释**易踩坑（需保证 PC1 与"绿色低碳"同向）；n=10 < 经验阈值，PCA 不稳；新变量物理含义模糊（Judge 要求"降维后解释新变量"） |
| W4 标准差法 | 权重 ∝ 各列标准差 | 否 | 是（最强离散度驱动） | 最简单 | 与熵权同病且更糙，仅备选 |
| **W0 等权**（基线） | 全 1/7 | **隐性放大 D1**（簇 4 项各占 1/7=簇占 4/7） | 否 | 零主观、零信息、最透明 | 重复计权最严重；作"零信息基线"参照 |
| W5 主客观组合（加分备选） | CRITIC × 领域先验（如碳/能耗政策权重）几何/线性融合 | 看主观侧 | — | 缓解 A7（客观未必符合政策重要性，如碳指标政策更重但可能被赋低权） | 引入主观即削弱"客观"硬约束，须显式声明 + 敏感性对照，**不作主方案** |

**A7 立场（客观 vs 领域先验）**：
- README 硬性要"客观、可复现"→ **主方案纯客观（CRITIC）**，不注入人为权重，保住可复现底线。
- 但纯客观的代价（碳指标政策更重却可能被数据赋低权）是真问题 → **在 Q3 用 W5 组合赋权做一次"敏感性对照"**，量化"若按政策抬高碳/能耗权重，排名翻不翻"，把 A7 张力**显式摆给决策者**而非藏起来。这是把局限转成加分（Judge §1 铁律 5："最强队把假设当敏感性来回访"）。

**D1 处理立场（Scout 头号埋点，必须显式回应）**：
- **主方案 W1 CRITIC 用冲突性项自动给 D1 簇降权**（簇内两两 |r|≥0.88，冲突性 Σ(1−r) 小 → 权重被压），这是**相关性校正赋权**路线。
- **对照方案 W3 PCA 用线性去相关**把簇内冗余合并进主成分，这是**降维去冗余**路线。
- **反面镜子 W2 熵权 / W0 等权**故意不处理 D1，留作"去冗余是否改变结论"的对照——**这正是 Critic 该攻击的命门#1**（见第三节）。
- 三条路线（校正赋权 / 降维 / 不处理）横向比，正面兑现 Router"≥2 套赋权方案"与 Scout "至少一套显式处理 D1"的硬要求。

### 阶段 4 · 聚合与排序

| 候选 | 做法 | 对短板的态度 | 优 | 劣 / 风险 |
|---|---|---|---|---|
| **T1 加权 TOPSIS**（主） | 算到正/负理想解的加权欧氏距离，贴近度 `C = D⁻/(D⁺+D⁻)` | **对短板敏感**（离负理想解近会被惩罚，高分项难完全补偿短板） | 经典 MADM、可解释（"离最优有多远"）、与 Q2 短板诊断天然衔接 | 距离对权重与无量纲化口径都敏感；贴近度非线性，分差解释须克制（A6） |
| T2 加权和 SAW（对照） | `score = Σ wⱼ·rⱼ` 线性加权求和 | **可被高分补偿**（一项突出能补另一项垫底） | 最简单、最透明、分解性最好（Q2 可直接做指标贡献分解） | 短板被掩盖；与 TOPSIS 给不同名次 → Q3 聚合范式敏感性的主对照 |
| T3 加权积 WPM | `score = Π rⱼ^wⱼ` | **对短板最敏感**（任一项趋 0 拉垮总分） | 短板惩罚最强 | 需 r>0（与 N2 z-score 负值不兼容）；对小值过度敏感 |
| T4 灰色关联 GRA | 与参考序列的关联度 | 中等 | 小样本稳健、对分布无假设 | 分辨系数 ρ 是隐藏自由度、可解释性弱 |

**立场**：主聚合 T1（TOPSIS，对短板敏感、与 Q2 衔接好）；**对照 T2 加权和**（线性补偿，做"聚合范式是否改变名次"的敏感性，Analyst 陷阱#5 的兑现）。Q2 的指标短板归因优先用 **T2 的可加分解**（每地区每指标对得分缺口的贡献 = wⱼ·(r*ⱼ−rᵢⱼ)），因为加权和分解性最干净；TOPSIS 侧用"到负理想解各维分量"交叉佐证。

## 二、选定方案（主 + 对照，全部喂 Q3）

| 代号 | 无量纲化 | 赋权 | 聚合 | 角色 | 关键卖点 |
|---|---|---|---|---|---|
| **M1（主方案）** | min-max | **CRITIC** | 加权 TOPSIS | 主排名来源 | 显式处理 D1（冲突性降权）+ 抗短板补偿 |
| **M2（首要对照）** | min-max | 熵权法 | 加权和 | 反面镜子 | **不处理 D1** + 离散度驱动 + 线性补偿，对比"去冗余是否改名次" |
| **M3（去冗余对照）** | min-max | PCA | 主成分得分 | 独立验证 | 线性去相关，独立验证 D1 去冗余结论 |
| **M0（零信息基线）** | min-max | 等权 1/7 | 加权和 | 基线 | 评委要"baseline 必须被对照"（winning §3C/§3E） |
| M5（A7 对照，加分） | min-max | CRITIC×政策先验 | 加权 TOPSIS | 张力对照 | 量化"抬高碳/能耗政策权重后排名翻不翻" |

**主排名 = M1**；M0/M2/M3 三套与 M1 做 Spearman/Kendall 秩相关，量化"换赋权/换聚合/去冗余"的排名翻动（Q3 核心）；M5 量化 A7 张力（Q3 加分）。

**模型完整定义（M1）**：
- 变量：标准化决策值 `rᵢⱼ ∈ [0,1]`（i 地区, j 指标），权重 `wⱼ`（Σwⱼ=1）。
- 参数：各列 min/max（来自 N1）；CRITIC 中各列标准差 σⱼ 与相关矩阵 R（来自转向后数据）。
- 赋权：`Cⱼ = σⱼ · Σₖ(1−r_{jk})`，`wⱼ = Cⱼ / ΣₘCₘ`。
- 加权决策值 `vᵢⱼ = wⱼ·rᵢⱼ`；正理想 `v⁺ⱼ=maxᵢvᵢⱼ`，负理想 `v⁻ⱼ=minᵢvᵢⱼ`。
- 距离 `D⁺ᵢ=√Σⱼ(vᵢⱼ−v⁺ⱼ)²`，`D⁻ᵢ=√Σⱼ(vᵢⱼ−v⁻ⱼ)²`；贴近度 `Cᵢ=D⁻ᵢ/(D⁺ᵢ+D⁻ᵢ)`。
- 排序：`Cᵢ` 降序即排名。约束：确定性（无随机种子）、字节级可复现。

## 三、PoC 冒烟点（≤30 行，求解前必跑，跑通+量级合理才正式选用）

> 借鉴 G2 选用前冒烟闸：先验证"模型跑得动 + 数值量级合理 + 可手验子结论"，否则退回换方法（接 CP2）。
> **可手验的边界子结论**：Scout §5.1 显示 **G 在"能源-碳-减排"簇 4 项全部最优**、**H 在森林/公园/PM2.5 三项领先**、**F 在 5 项垫底**。
> 因此**任何合理方案都应给出 G 或 H 排名第 1 梯队、F 垫底**——若 PoC 跑出 F 不垫底或 G/H 不靠前，说明方向转反或量纲化口径错（D4），立即退回。

```python
import numpy as np, pandas as pd
df = pd.read_csv(r"E:\Code\qoder\cases\practice_04_green_eval\data\indicators.csv", index_col="region")
rev = ["energy_intensity","carbon_intensity","pm25"]            # 3 逆向
# 阶段1+2: min-max 同口径转向(逆向 max-x, 正向 x-min)
R = df.copy().astype(float)
for c in df.columns:
    lo, hi = df[c].min(), df[c].max()
    R[c] = (df[c]-lo)/(hi-lo) if c not in rev else (hi-df[c])/(hi-lo)
# 阶段3: CRITIC 权重 (标准差 × 冲突性 Σ(1-r))
sd = R.std(ddof=1); corr = R.corr()
conflict = (1-corr).sum()
w = (sd*conflict); w = w/w.sum()
# 阶段4: 加权 TOPSIS
V = R*w; vp, vn = V.max(), V.min()
Dp = np.sqrt(((V-vp)**2).sum(1)); Dn = np.sqrt(((V-vn)**2).sum(1))
C = Dn/(Dp+Dn)
print("weights:\n", w.round(3))
print("TOPSIS closeness (desc):\n", C.sort_values(ascending=False).round(3))
# 冒烟断言: F 垫底, G/H 进前2 (Scout §5.1 边界子结论)
rank = C.rank(ascending=False)
assert rank["F"] >= 9, "F 应垫底, 否则方向/口径错"
assert min(rank["G"], rank["H"]) <= 2, "G 或 H 应居首梯队"
print("SMOKE PASS: F 垫底 & G/H 居首梯队, 方向口径自洽")
```

**冒烟通过判据**（三条同时满足才正式选用 M1）：
1. **跑得通**：无报错、无 NaN（min-max 分母不为 0，Scout 已证各列 max>min）。
2. **量级合理**：权重 Σw=1 且各 wⱼ∈(0,0.3) 无单指标霸权；贴近度 Cᵢ∈(0,1)。
3. **手验子结论**：**F 垫底、G/H 居首梯队**（Scout §5.1 边界事实，与方向转向自洽）；CRITIC 应把 D1 簇（energy/carbon 等）权重压到低于其单独标准差排名（验证冲突性项生效）。

> 若断言失败：优先怀疑 D4 方向转反 / N1 口径不自洽 → 退回阶段 1–2 重查；若权重出现单指标 >0.5 霸权 → 怀疑赋权被极端列主导，退回阶段 3 换 W3/组合。**接 CP2 回退**。

**三大命门小结**（交 Critic 集火）：
