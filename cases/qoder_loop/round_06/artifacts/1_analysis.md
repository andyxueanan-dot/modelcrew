# Analysis — 碳税政策多目标分析

## 1. 问题分解

### Q1: 场景分析（τ=200）
- **核心**: 给定税率下各产业的减排量、GDP 损失、就业损失
- **输出**: 4 产业的详细影响 + 省级汇总

### Q2: 目标倒推（碳排放减半）
- **核心**: 求解使总排放从 5 亿吨降到 2.5 亿吨的最低 τ
- **方法**: 二分搜索 τ ∈ [0, 1000]，目标函数 f(τ) = 总排放(τ) - 2.5 亿吨
- **输出**: τ_min + 对应的 GDP 和就业损失

### Q3: 绿色补贴的净就业效果
- **核心**: 碳税收入 → 绿色补贴 → 新增就业 → 净就业 = 新增 - 损失
- **输出**: "就业中性"碳税（净就业 = 0 的 τ）

## 2. 假设清单

| # | 假设 | 理由 |
|---|------|------|
| A1 | 减排弹性恒定（不随 τ 变化） | 简化模型，实际中弹性可能递减 |
| A2 | GDP 损失 = 碳税成本 × 0.3 | 假设 30% 转嫁为产出下降，70% 转嫁为消费者价格上升 |
| A3 | 就业损失 = GDP 损失 / 劳动生产率 | 忽略劳动力市场摩擦（再培训、迁移成本） |
| A4 | 绿色补贴每 1 亿元创造 500 个岗位 | 假设绿色岗位与损失岗位可比 |
| A5 | 碳税收入全部用于绿色补贴 | 忽略行政成本和漏损 |

## 3. 模型方程

### 减排
ΔE_i(τ) = E_i × (1 - exp(-ε_i × τ / 100))

### 碳税成本
Cost_i(τ) = τ × (E_i - ΔE_i(τ))

### GDP 损失
ΔGDP_i(τ) = Cost_i(τ) × 0.3

### 就业损失
ΔJob_i(τ) = ΔGDP_i(τ) / (GDP_i / Job_i)

### 碳税收入
Revenue(τ) = Σ Cost_i(τ)

### 绿色就业
GreenJobs(τ) = Revenue(τ) × 500 / 10^8

### 净就业
NetJobs(τ) = GreenJobs(τ) - Σ ΔJob_i(τ)

## 4. 预期陷阱

### 陷阱 1: 电力产业的"高弹性优势"
电力产业排放最高（2.5 亿吨），但弹性也最高（0.15）。高碳税下电力的减排贡献最大，但 GDP 损失也最大（因为碳税成本高）。

### 陷阱 2: 农业的"低弹性困境"
农业排放低（0.4 亿吨），但弹性极低（0.03）。即使高碳税，农业减排也很小，但就业损失可能大（因为劳动生产率低）。

### 陷阱 3: "就业中性"碳税的存在性
绿色补贴创造的岗位是否与损失的岗位匹配？如果绿色岗位需要不同技能，可能存在结构性失业。

## 5. 方法选择理由

### 为什么用场景分析？
- 政策分析的标准方法
- 直观展示不同税率的效果
- 便于决策者理解

### 为什么用 Pareto 前沿？
- 展示减排 vs GDP 损失的权衡
- 识别"最优"税率（Pareto 最优解）

### 为什么用二分搜索？
- 目标函数 f(τ) 单调递减（τ 越高排放越低）
- 简单可靠，无需导数
