# Analysis — 城市应急疏散网络

## 1. 网络特征

- 10 节点 (3S + 4M + 3T), 20 条有向边
- 边属性: 时间(3-12min) + 容量(100-400人/h)
- S3 出度最低(2条), M2 入度最高(4条: S1→M2, M1→M2, S2→M2, M3→M2)

## 2. 预期发现

- S1→T1 最短路径: S1→M1→T1(15min), 但 M1 入容量仅450
- S1→M2→T1(15min) 是等时替代路径, 容量更充裕
- M2 是最关键枢纽(6条边), 移除后 S3→T1 断开
- S3→T1 路径最长(21min, 3跳), 边连通度最低(=1)

## 3. 方法论完整度(回应R4教训)

| R4 扣分点 | R10 是否覆盖 |
|-----------|------------|
| 缺 Yen's k-最短路径 | ✅ K=3 |
| 缺顶点连通度 κ | ✅ κ=2 |
| 缺边连通度 λ | ✅ λ=1 |
| 缺多节点失效 | ✅ M1+M3 |
