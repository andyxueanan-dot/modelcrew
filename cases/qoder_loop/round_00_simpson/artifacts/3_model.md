# Model Definitions — 派单策略 A/B 评估

## 候选方法

| # | 方法 | 优点 | 缺点 | 适用场景 |
|---|------|------|------|---------|
| **M1** | **双比例 z 检验（汇总 + 分层）** | 经典、简单、教科书级；直接给出 p 值和置信区间；评委一看就懂 | 不控制混杂变量；无法给出"校正后"的统一效应量 | 快速判断各层差异是否显著 |
| **M2** | **Mantel-Haenszel 方法** | 专为辛普森悖论设计；控制混杂变量后给出共同 odds ratio；有标准检验 (Breslow-Day) | 假设各层 OR 同质（可用 Breslow-Day 检验）；对小样本层不够稳健 | 控制难度混杂后比较策略能力 |
| **M3** | **Bayesian Beta-Binomial** | 自然处理小样本（B-高难度 n=50）；给出后验分布和可信区间；可计算"B 优于 A 的概率" | 需要先验选择；计算较复杂；评委可能不熟悉 | 小样本层的精确推断 |

## 推荐方案：M1 + M2（互补）

- **M1 (z 检验)** 作为 baseline：汇总和分层各做一次，给出 p 值和 95% CI
- **M2 (Mantel-Haenszel)** 作为核心：控制难度混杂后的统一效应量估计
- M3 留作 sensitivity analysis（如时间允许）

## 具体模型

### 模型 1：双比例 z 检验

**汇总层面：**
```
H₀: p_A = p_B (两策略按时率相同)
H₁: p_A ≠ p_B

p̂_A = 290/350 = 0.8286
p̂_B = 182/350 = 0.5200
p̂_pool = (290+182)/(350+350) = 472/700 = 0.6743

z = (p̂_A - p̂_B) / √(p̂_pool(1-p̂_pool)(1/n_A + 1/n_B))
```

**分层层面（每层独立 z 检验）：**
```
低难度层: H₀: p_A_low = p_B_low
高难度层: H₀: p_A_high = p_B_high
```

### 模型 2：Mantel-Haenszel

**2×2×K 列联表（K=2 层）：**

层 k 的 odds ratio:
```
OR_k = (a_k × d_k) / (b_k × c_k)

其中：
a_k = A 按时数, b_k = A 未按时数
c_k = B 按时数, d_k = B 未按时数
```

MH 合并 odds ratio:
```
OR_MH = Σ(a_k d_k / N_k) / Σ(b_k c_k / N_k)
其中 N_k = a_k + b_k + c_k + d_k
```

MH 检验统计量:
```
χ²_MH = (|Σ a_k - Σ E(a_k)| - 0.5)² / Σ Var(a_k)
```

### 模型 3：标准化率（直接法）

以全体 700 单为标准人口：
- 标准低难度比例: 350/700 = 50%
- 标准高难度比例: 350/700 = 50%

标准化率:
```
p_A_std = 0.50 × p̂_A_low + 0.50 × p̂_A_high
p_B_std = 0.50 × p̂_B_low + 0.50 × p̂_B_high
```

## 待审计结论

| # | 结论 | Confidence | Depends On |
|---|------|-----------|-----------|
| C1 | 汇总层面 A 的按时率 (82.9%) 显著高于 B (52.0%) | High（数学事实） | — |
| C2 | 分层后每一层 B 的按时率都不低于 A（低难度：94%>90%；高难度：需统计检验） | Medium | A4（样本量） |
| C3 | 辛普森悖论成立：汇总趋势与分层趋势方向相反，原因是订单难度分配不均 | High | A1, A3 |
| C4 | 控制难度后，策略 B 的"内在能力"优于策略 A（MH OR > 1 favoring B） | Medium | A2, A3, A4 |
| C5 | 全市推广建议：应根据订单难度差异化分配策略 | Medium | A5, C4 |
| C6 | 结论对分层口径敏感：若重新定义难度阈值，结论可能改变 | Low | A1 |
