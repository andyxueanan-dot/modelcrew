# Analysis — 供应链网络韧性分析

## 1. 问题分解

### Q1: 最短路径 (S1→R1)
- **核心**: Dijkstra 算法在加权有向图上求最短路径
- **输出**: 路径序列、总成本、路径可靠性
- **复杂度**: O((V+E)log V) with priority queue

### Q2: 关键节点识别
- **核心**: 三种中心性指标
  - 度中心性 (Degree): 直接连接数
  - 介数中心性 (Betweenness): 控制最短路径的能力
  - 接近中心性 (Closeness): 到其他节点的平均距离
- **输出**: 12 节点的中心性排名，识别"瓶颈"节点

### Q3: 节点失效模拟
- **核心**: 移除 M1 后重新计算 S1→R1 的最短路径
- **输出**: 替代路径、成本变化、可靠性变化
- **延伸**: 网络连通性分析（移除 M1 后是否有断连节点）

## 2. 假设清单

| # | 假设 | 理由 |
|---|------|------|
| A1 | 图是有向的（运输方向固定） | 供应链流向：供应商→制造商→分销→零售 |
| A2 | 边权重是静态的（成本不随流量变化） | 简化模型，不考虑拥堵效应 |
| A3 | 节点失效是完全失效（非部分降级） | 简化模型，二元状态 |
| A4 | 路径可靠性 = 所有边可靠性乘积 | 假设边失效独立 |
| A5 | 节点容量足够（不考虑容量约束） | 聚焦网络拓扑，不考虑流量分配 |

## 3. 数据需求

### 输入
- 节点列表（12 个）
- 边列表（17 条）：起点、终点、成本、可靠性、容量
- 源节点 S1，目标节点 R1

### 输出
- S1→R1 最短路径（节点序列 + 总成本 + 可靠性）
- 12 节点的中心性指标
- M1 失效后的替代路径

## 4. 预期陷阱

### 陷阱 1: M1 可能是高介数节点
M1 连接 S1/S2（供应商）到 D1/D2（分销），可能是多条最短路径的必经之路。如果 M1 失效，S1→R1 的成本可能大幅上升。

### 陷阱 2: 最短路径 ≠ 最可靠路径
最短路径（成本最低）可能经过低可靠性的边。需要同时报告成本和可靠性，让用户权衡。

### 陷阱 3: 节点失效的级联效应
移除 M1 不仅影响 S1→R1，还可能影响 S2→R1 等其他路径。需要分析整个网络的连通性变化。

## 5. 方法选择理由

### 为什么用 Dijkstra？
- 边权重非负（成本 ≥ 0）
- 单源单目标最短路径
- 复杂度低，易于实现

### 为什么用三种中心性？
- 度中心性：识别"hub"节点（连接多）
- 介数中心性：识别"瓶颈"节点（控制流量）
- 接近中心性：识别"核心"节点（距离近）
- 三者互补，单一指标可能误导

### 为什么模拟 M1 失效？
- M1 连接 4 条边（S1→M1, S2→M1, M1→D1, M1→D2），度中心性高
- 预期 M1 是介数中心性最高的节点
- 失效模拟能直观展示网络韧性
