# 案例·应急调度策略评估（数据型 · 自纠 demo）

> 本案例由 **ModelCrew 子代理端到端真跑**，用于演示"系统自己跑 → Critic 抓错 → 自我更正"的闭环。
> 数据由 `data/generate_data.py` 确定性生成（无随机、字节级可复现）。

## 赛题（自拟，数据型）
某市试点两套应急调度派遣策略 **A** 与 **B**，在一段时期内记录了每次出警的：
- `policy`：本次出警采用的策略（A / B）
- `severity`：呼叫严重度（Low 轻 / High 重）
- `on_time`：是否在目标时限内到达（1 是 / 0 否）

**要回答：**
- **Q1**：哪套策略的"按时到达率"更高？应向市政推荐哪一套？
- **Q2**：这个结论可靠吗？是否存在混杂导致的误判？
- **Q3**：若要公平比较两套策略，正确的做法是什么？给出可落地建议。

## 数据
`data/dispatch.csv`（700 行，由 `data/generate_data.py` 确定性生成）。字段如上。
