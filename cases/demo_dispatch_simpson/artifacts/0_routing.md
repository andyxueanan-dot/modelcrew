## 1. 题型分类结论

| 维度 | 结论 | 依据（引用题面） |
|---|---|---|
| 主类型 | **数据型**（≈ 美赛 C / 国赛 C） | README 标题即"数据型 · 自纠 demo"；Q1"哪套策略的按时到达率更高"是基于观测数据的**统计比较**；数据为 `dispatch.csv` 700 行三字段 `policy/severity/on_time` 的**记录型表格数据** |
| 驱动方式 | **数据驱动**（无机理方程） | 没有物理过程/微分方程/动力学；要回答的是从样本里**估计率、判断可靠性**，纯由数据说话 |
| 隐藏副标签 | **因果推断 / 混杂（confounding）** | Q2"是否存在混杂导致的误判"、Q3"公平比较两套策略的正确做法"——数据里 `severity` 是潜在**混杂变量**，几乎可以确定要处理 **Simpson 悖论**（案例目录名 `demo_dispatch_simpson` 也暗示这一点） |

**一句话定性**：这是一道**数据型 + 因果/混杂识别**题，核心不是"算个率"，而是"识破合并率的陷阱、给出分层/标准化后的可信结论"。题面三问层层递进（描述 → 质疑 → 修正）正是典型的数据型自纠结构。

> 存疑标注：题型本身不存疑（数据驱动明确）。唯一需 Critic 重点把关的是 **Q2 的混杂判定**——必须由数据实证（分层后率是否反转），不能仅凭直觉断言，故下游安排 Critic 硬闸门专盯"过拟合/随机性/混杂误判"。

## 2. 召唤策略（本次召唤了谁、为什么）

按"数据型"标准链路召唤，并在末端统一接 **Writer(含 LaTeX) → Judge**。

| 顺序 | 子代理 | 召唤理由（针对本题） |
|---|---|---|
| 1 | `modelcrew-analyst` | 审题、明确 Q1–Q3 与变量角色（处理变量 policy、结果 on_time、疑似混杂 severity）、列研究假设 |
| 2 | `modelcrew-scout`（**重**） | 数据型必召的探索环节：算合并按时率、**按 severity 分层**算率、画交叉表，第一时间暴露是否发生率反转（Simpson） |
| 3 | `modelcrew-modeler` | 选统计模型：分层分析 / 标准化率（standardization）/ 卡方或 logistic（policy + severity）等，给"公平比较"的正式方法 |
| 4 | `modelcrew-solver` | 在数据上真跑出分层率、标准化率、模型系数与置信区间，给可复现数字 |
| 5 | `modelcrew-critic`（**硬闸门**，盯随机性/过拟合/混杂） | 审计：合并结论是否被混杂污染、分层结论是否稳健、样本量是否够、有无过度解读；触发回退最多 2 轮 |
| 6 | `modelcrew-writer` | 含 LaTeX 成稿：把"合并率→分层反转→标准化修正"的因果故事写清，给市政可落地建议 |
| 7 | `modelcrew-judge` | 终局按 `rubrics.md` 打分、预估奖级、给"只改一处"建议（L3 五视角 Panel） |

**未召唤**：连续型的物理假设/微分方程/数值稳定性专项不适用（无机理）；优化型的规划/启发式不适用（无决策最优化目标，"推荐哪套"是统计判定而非求最优解）。

## 3. 协作时序图（含 4 个人把舵卡点）

```
Analyst（审题·变量角色·假设）
   │
   ├─[🛑CP1 审题确认]── 人拍板：是否认可"severity 为疑似混杂、Q2 要查 Simpson"的定题
   │
Scout（重：合并率 + 按 severity 分层率 + 交叉表，暴露反转）
   │
Modeler（分层分析 / 标准化率 / logistic 公平比较方法）
   │
   ├─[🛑CP2 选模型确认]── 人拍板：用分层标准化还是回归调整作为主结论方法
   │
Solver（真跑：分层率、标准化率、系数 + 置信区间，可复现）
   │
Critic（硬闸门：混杂是否被正确处理 / 是否过拟合 / 样本量 / 是否过度解读）
   │
   ├─[🛑CP3 若 Critic 出❌]── 人拍板是否回退重做（Modeler/Solver，最多 2 轮）
   │        └──❌──► 回退 ──► 重跑 ──► 再过 Critic
   │
Writer（含 LaTeX：合并→反转→标准化的因果叙事 + 市政建议）
   │
Judge（按 rubrics 打分 / 预估奖级 / 只改一处）
   │
   └─[🛑CP4 评委复盘]── 人拍板：采纳 Judge 建议后定稿
```

## 5. 铁律备注
- 准确性高于流畅性：本题"按时率谁高"的合并结论极可能与分层结论相反，**严禁**让 Writer 直接采用合并率结论；Critic 必须用分层实证把关后才放行。
- 题型不存疑；唯一需多方案比对处为"公平比较方法"（分层标准化 vs 回归调整），交由 Modeler 给两套、Critic 比较择优。
