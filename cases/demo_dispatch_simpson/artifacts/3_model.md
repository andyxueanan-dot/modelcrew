## 1. 记号与公共定义（三问共用）

- 处理 $T \in \{A, B\}$（policy）；混杂 $S \in \{L=\text{Low}, H=\text{High}\}$（severity）；结果 $Y \in \{0,1\}$（on_time）。
- 单元计数 $n_{ts}$ = 策略 $t$、层 $s$ 的样本数；成功数 $y_{ts}$；层内按时率 $\hat p_{ts} = y_{ts}/n_{ts}$。
- 由 Scout §3 锁定的数（本节直接引用，不重算）：

| | $n_{tL}$ | $\hat p_{tL}$ | $n_{tH}$ | $\hat p_{tH}$ | 合并率 $\hat p_t$ |
|---|---|---|---|---|---|
| A | 300 | 0.9000 | 50 | 0.4000 | 0.8286 |
| B | 50 | 0.9400 | 300 | 0.4500 | 0.5200 |

- 层边际权重（用于标准化）：$N_L = n_{AL}+n_{BL}=350$，$N_H=350$，总 $N=700$。

## 2. Q1 模型 · 边际（合并）率比较 —— 基线，疑误判

### 2.1 模型定义
- **统计量**：合并按时率 $\hat p_A = \dfrac{\sum_s y_{As}}{\sum_s n_{As}}$，$\hat p_B$ 同理；比较 $\hat p_A$ vs $\hat p_B$。
- **检验**：两比例差 $z$ 检验（或等价 $2\times2$ 卡方）。
  $$ z = \frac{\hat p_A - \hat p_B}{\sqrt{\hat p(1-\hat p)(1/N_A + 1/N_B)}},\quad \hat p=\frac{y_A+y_B}{N_A+N_B} $$
- **变量**：仅 policy。**约束**：无（描述性比较）。

### 2.2 为什么选它 / 替代 / 代价
- **为什么**：这是 README Q1 字面问的口径，必须先给出作为"陷阱基线"——展示直觉答案，才能在 Q2 推翻它，叙事才完整。
- **替代**：直接给率差点估计而不做检验（更弱，舍）。
- **代价（关键）**：**完全忽略 severity**，在 T2（A 偏轻案、B 偏重案，6 倍失衡）下，$\hat p_A-\hat p_B$ 把"案件难度差"误算成"策略效果差"。→ **此结论标记"待审计/疑混杂误判"，严禁作为推荐依据。**

## 3. Q2 模型 · 分层比较 + Simpson 反转检测

### 3.1 模型定义
- **层内率**：$\hat p_{As}, \hat p_{Bs}$（$s\in\{L,H\}$），各带 Wilson/正态近似置信区间（区间由 Solver 算）。
- **Simpson 反转判定准则（可证伪）**：设合并方向 $d_0 = \operatorname{sign}(\hat p_A - \hat p_B)$，层内方向 $d_s = \operatorname{sign}(\hat p_{As}-\hat p_{Bs})$。
  $$ \textbf{反转成立} \iff d_0 \neq d_L \;\textbf{且}\; d_0 \neq d_H $$
  即"合并方向"与**每个层内方向**都相反。（Scout §3.5 数值：$d_0=+$(A高)，$d_L=d_H=-$(B高) → 反转成立。本节把它形式化，Solver 复算确认。）
- **混杂判定三条件**（把 Analyst A1 / Scout T1 写成可检准则）：
  1. $S$ 关联 $T$：$P(S=H\mid A)=0.143 \neq P(S=H\mid B)=0.857$ ✔（差 6 倍）。
  2. $S$ 关联 $Y$（给定 $T$ 内仍成立）：$P(Y\mid L)=0.906 \neq P(Y\mid H)=0.443$ ✔（差 0.46）。
  3. $S$ 非 $T\to Y$ 的中介（机制假设）：severity 是"案件固有难度"，先于派遣结果 → 假设为混杂而非中介（依赖 A5/T6，**待核实**）。

### 3.2 为什么选它 / 替代 / 代价
- **为什么**：分层是检验混杂"是否真导致误判"的最直接、最可解释手段；反转准则把"是否误判"变成 0/1 可判定命题，可证伪。
- **替代**：直接上回归交互项检验异质性（信息更多但更难讲，留作 Q3 佐证）。
- **代价**：层内最小格 $n=50$（High-A、Low-B），分层率有抽样波动 → **必须带 CI**，否则反转可能是噪声（T4/C4）。Solver 须给每层 CI，Critic 查反转在 CI 下是否稳健。

## 4. Q3 模型 · 公平比较的三候选（CP2 重点）

> 三者都"控制 severity 后比较 A/B"，但**估计量、可解释性、对小格/外推的敏感度不同**。下面逐一定义 + 取舍。

### 候选 (a) 直接标准化率差（Directly Standardized Rate Difference）
- **定义**：选一套**统一层权重** $w_s$（$w_L+w_H=1$），各策略的标准化率
  $$ \hat p_t^{\text{std}} = \sum_{s\in\{L,H\}} w_s\,\hat p_{ts},\qquad \Delta^{\text{std}} = \hat p_A^{\text{std}} - \hat p_B^{\text{std}} $$
- **权重方案**（敏感性必做，对应 C6）：
  - 总体权 $w_s=N_s/N=(0.5,0.5)$ → $\hat p_A^{\text{std}}=0.5(0.90)+0.5(0.40)=0.650$；$\hat p_B^{\text{std}}=0.5(0.94)+0.5(0.45)=0.695$；$\Delta^{\text{std}}=-0.045$（**B 高**）。
  - A 人群权 $(300/350, 50/350)$、B 人群权 $(50/350,300/350)$ 作为上下界敏感性。
- **优**：尺度直观（"百分点差"，市政易懂）；率差是政策语言。**劣**：结论随 $w_s$ 漂移（C6），需报多套权重；小格率方差进入加权，CI 偏宽。
- **量级预判**（待 Solver 证）：总体权下约 **−4.5 个百分点，B 略优**。

### 候选 (b) Mantel–Haenszel 合并比值比（MH-OR）
- **定义**：各层 $2\times2$ 表（行=策略，列=on_time），层内 OR 加权合并
  $$ \widehat{OR}_{MH} = \frac{\sum_s (a_s d_s / n_s)}{\sum_s (b_s c_s / n_s)} $$
  其中层 $s$ 表为 $\begin{smallmatrix}a_s & b_s\\ c_s & d_s\end{smallmatrix}$（$a$=A按时,$b$=A不按时,$c$=B按时,$d$=B不按时），$n_s$=层总数。
- **附带**：Breslow–Day 检验查"层间 OR 是否同质"（若两层 OR 差异大，合并 OR 失去意义，应改报分层）。
- **优**：流行病学标准混杂校正法，对小格更稳健，有成熟 CI（MH 方差）；单一数概括"控制 severity 后 A 相对 B 的优势比"。**劣**：OR 比率差不如百分点直观，需向市政翻译；假设层间效应同质。
- **量级预判**（待 Solver 证）：$OR_{MH}<1$ 表示 A 按时几率低于 B（即 **B 优**），预期落在约 0.7–0.9 区间，方向应与 (a) 一致。

### 候选 (c) Logistic 回归（on_time ~ policy + severity）
- **定义**：$\operatorname{logit} P(Y=1) = \beta_0 + \beta_1\mathbb{1}[T=B] + \beta_2\mathbb{1}[S=H]$。
  - $\beta_1$ = 控制 severity 后 B 对 A 的对数几率差；$e^{\beta_1}$ 为校正 OR（与 MH-OR 应接近）。
  - 加分变体：加交互项 $\beta_3\mathbb{1}[T=B]\mathbb{1}[S=H]$ 检验效应异质性（呼应 Breslow–Day）。
- **优**：可扩展（未来加协变量即插即用）、直接给 $\beta_1$ 的 Wald CI 与 $p$ 值、与 MH 互为印证。**劣**：本题仅 2 协变量时相对 MH 是"杀鸡用牛刀"；对模型设定（连接函数/交互）有依赖；系数解释需转成 OR 再转百分点，沟通成本高。
- **量级预判**（待 Solver 证）：$e^{\beta_1}>1$（B 几率高于 A），与 MH-OR 互为印证（注意 MH 是 A/B 编码，logistic 是 B/A 编码，方向需对齐再比）。

### 4.x 三候选横向取舍表

| 维度 | (a) 标准化率差 | (b) MH-OR | (c) Logistic |
|---|---|---|---|
| 输出尺度 | 百分点差（最直观） | 比值比 | 系数→OR |
| 市政可解释性 | ★★★ | ★★ | ★★ |
| 小格/稳健性 | 中（方差进加权） | ★★★（标准稳健法） | 中 |
| 权重/设定敏感 | 高（C6 权重漂移） | 低（同质假设） | 中（设定+交互） |
| 可扩展性 | 低 | 低 | ★★★ |
| 统计成熟度 | ★★ | ★★★ | ★★★ |
| 是否需配套检验 | 多权重敏感性 | Breslow–Day | 交互项/拟合优度 |

> **Modeler 倾向**：以 **(a) 直接标准化率差（总体权）为主结论**（市政语言最直观），**(b) MH-OR 作稳健性印证**，**(c) logistic 作可扩展性佐证 + 给交互检验**。三者**方向必须一致**（都指向 B 优）才放行——这是 Critic 的交叉验证点。最终交 CP2 拍板。

## 5. 🔥 选用前 PoC 冒烟闸（交 Solver 跑，≤30 行，正式选用前置）

> **目的**：防"纸上选了个跑不出来/量级离谱的模型"。下面这段必须**先跑通**，且数值满足验收标准，三候选才正式入选；任一不通过 → 退回换方法（接 CP2/CP3）。

```python
# PoC: 用 Scout 锁定的聚合数验证三候选可跑通 + 量级合理 + 方向一致 (<=30 行)
import numpy as np
# 层内: (A按时, A总, B按时, B总)  来自 2_data_brief §3.3
L = dict(yA=270, nA=300, yB=47,  nB=50)   # Low
H = dict(yA=20,  nA=50,  yB=135, nB=300)  # High
# --- 基线: 合并率 (Q1) ---
pA = (L['yA']+H['yA'])/(L['nA']+H['nA']); pB=(L['yB']+H['yB'])/(L['nB']+H['nB'])
print("Q1 合并率  A=%.4f B=%.4f  (期望 A>B, 0.829 vs 0.520)"%(pA,pB))
# --- (a) 标准化率差, 总体权 wL=wH=0.5 (Q3) ---
pAL,pAH = L['yA']/L['nA'], H['yA']/H['nA']; pBL,pBH=L['yB']/L['nB'],H['yB']/H['nB']
sA=0.5*pAL+0.5*pAH; sB=0.5*pBL+0.5*pBH
print("(a) 标准化率 A=%.4f B=%.4f  diff(A-B)=%.4f  (期望 <0, B优~ -0.045)"%(sA,sB,sA-sB))
# --- (b) MH-OR: 行=策略 列=on_time(1,0); A vs B ---
def cell(d): return d['yA'],d['nA']-d['yA'],d['yB'],d['nB']-d['yB']  # a,b,c,d
num=den=0.0
for s in (L,H):
    a,b,c,d=cell(s); n=a+b+c+d; num+=a*d/n; den+=b*c/n
print("(b) MH-OR(A/B)=%.4f  (期望 <1 即 B优, ~0.7-0.9)"%(num/den))
# --- 方向一致性闸 ---
ok = (pA>pB) and (sA-sB<0) and (num/den<1)
print("方向一致性: 合并A高 且 校正后B优 ->", "PASS 反转确认" if ok else "FAIL 需排查")
```

**验收标准（Solver 须逐条回报）**：
1. **能跑通**：无异常、无除零、无 NaN。
2. **量级合理**：合并率 A≈0.829 / B≈0.520；标准化率差 ≈ −0.045（±0.01）；MH-OR ∈ (0.5, 1.0)。
3. **方向一致**：合并口径 A 高，**而 (a)、(b) 校正后均 B 优** → 打印 `PASS 反转确认`。
4. 若任一失败（如 MH-OR>1 或标准化率差>0）→ **不放行**，退回检查数据引用/编码方向，触发 CP2 重议。

> PoC 通过后，Solver 再做"全量版"：补**置信区间**（层内 Wilson CI、标准化率差 CI、MH-OR CI）、**多权重敏感性**、**Breslow–Day / 交互项**、**logistic 完整拟合**。PoC 仅是入选闸，不替代全量求解。

# 🛑 CHECKPOINT · CP2 选模型

**背景**：Q1（合并率，A 高）、Q2（分层反转，两层 B 高）方法已定且 Scout 已实证；分歧只在 **Q3"公平比较"的主结论方法**。三候选方向应一致（都指向 B 优），区别在尺度/稳健性/可解释性/可扩展性。

- **候选 A = 直接标准化率差（总体权为主 + 多权重敏感性）**
  - 优：输出"百分点差"，市政最易懂，是政策语言；预判 B 优约 4.5 个百分点。
  - 劣：结论随权重漂移（C6），须报多套权重区间；小格方差进加权，CI 偏宽。
- **候选 B = Mantel–Haenszel 合并 OR（+ Breslow–Day 同质检验）**
  - 优：流行病学标准混杂校正法，对小格(n=50)最稳健，CI 成熟；单数概括校正后优势比。
  - 劣：比值比不如百分点直观，需向市政翻译；假设层间效应同质。
- **候选 C = Logistic 回归 on_time ~ policy + severity（+ 交互项）**
  - 优：直接给 $\beta_1$ 的 CI 与 $p$ 值，可扩展（未来加协变量即插即用），与 MH 互为印证。
  - 劣：本题仅 2 协变量是"杀鸡用牛刀"；解释需层层转译，沟通成本高。

- **我的倾向**：**A 为主结论**（市政可解释性最高、率差是政策语言），**B 作稳健性印证**（小格更稳、对冲权重漂移质疑），**C 作可扩展性佐证 + 交互检验**。三者方向一致才放行——这正好构成 Critic 的交叉验证铁证。
