# 反模式清单 anti_patterns

> 供 `modelcrew-critic`（审计对照）与 `modelcrew-writer`（写作自查）。
> 这些是数学建模论文反复出现的失分点。终审时**逐条对照，命中即修**。
> 每条配：现象 / 为何扣分 / 修复。

## A. 摘要类（权重最重，评委 30 秒筛选就看这里）
**A1 摘要无定量结果**
- 现象："得到了较好的结果""求解效果良好"
- 为何扣分：评委无法判断你的工作量与水平
- 修复：写成"最优配比 (0.42,0.31,0.27)，误差 1.8%，较基线提升 23%"

**A2 摘要不分段 / 段序错乱**
- 现象：一整段铺到底，或创新点埋在中间
- 修复：5 段式——背景→方法→关键结果(带数)→创新→结论，详见 `writing_templates.md`

**A3 摘要与正文不符**
- 现象：摘要说用 LSTM，正文却是 BP
- 修复：摘要**最后**写，基于定稿正文

**A4 关键词空泛**
- 现象：关键词只有"数学建模""优化""分析"
- 修复：≥2 个来自具体方法（如"熵权-TOPSIS"），≥2 个来自题目领域

## B. 模型类
**B1 假设无支撑 / 假设过强**
- 现象："假设市场完全理性"却不说为什么、也不评估代价
- 修复：每条假设写"为什么合理 + 若不成立影响什么"

**B2 直接上最复杂的模型**
- 现象：不做 baseline 就上深度学习
- 为何扣分：没有对比，无法证明复杂模型值得
- 修复：先简单 baseline，再复杂模型并量化提升

**B3 模型与问题脱节**
- 现象：套了个流行模型，但没回答题目问的子问题
- 修复：每个模型明确对应哪个子问题

**B4 符号/变量前后不一致**
- 现象：同一个量前文叫 $x_i$ 后文叫 $a_i$
- 修复：建一张符号表，全文统一（Critic 的 L2 一致性回检要查这个）

## C. 求解与结果类
**C1 相关当因果**
- 现象："X 与 Y 高度相关，所以 X 导致 Y"
- 修复：明确只说相关，或给因果识别策略（控制混杂/自然实验）

**C2 没有灵敏度分析**
- 现象：给一组参数得一个结果就结束
- 为何扣分：美赛 sensitivity 是独立大节，国赛也必查
- 修复：关键参数 ±5~10% 看结果稳不稳，作图

**C3 过拟合 / 数据泄漏**
- 现象：在全数据上调参、用未来信息预测过去
- 修复：严格 train/test 划分；时间序列用时序 CV 不随机打乱

**C4 结果没有物理意义**
- 现象：算出"最优温度 -300℃"也不察觉
- 修复：结果带单位 + 量级常识检查（边界/守恒）

**C4b 在统计平局/等价上宣布"赢家"**（蒸馏自 practice_01 Critic ⚠️）
- 现象：模型 A、B 指标差到小数点后好几位才不同（如 MAPE 差 <0.001pp）、甚至预测序列逐位相同，却写"A 胜出/A 更优"
- 为何扣分：与"把不显著当显著"同源的过度解读；评委一核对就发现"赢家"是噪声
- 修复：实质等价就写"实质等价"，取其一须给**与性能无关的理由**（可解释性/可扩展/稳健），并点明"等价本身是稳健性正面信号"。呼应 `inconclusive_playbook.md`——差异未达可判定阈值时不夸大方向

**C4c 把阈值外推到未被数据/冻结覆盖的"圆整数"**（蒸馏自 practice_02 Critic ⚠️）
- 现象：数据只测了 c=0.30/0.40（无解），却在结论写"力度不足 **50%** 无解"——50% 未被任何 frozen 数字覆盖，且 c=0.50 实际有解（τ_max=24）
- 为何扣分：圆整数读着顺口但是"凑"的，评委一复算边界就露馅；与摘要口径还可能不一致
- 修复：阈值只写**数据/冻结直接支持的那个值**（"≤40%"），需要给"舒适余量"时显式分开说（"≥50% 才有余量"），绝不让圆整数暗示一个未验证的分界

**C5 代码不可复现**
- 现象：没记随机种子、参数、环境
- 修复：固定 seed，记录全部运行设置

**C5b 重型范式规模失控 / 状态爆炸跑不完**（蒸馏自 practice_03 穿越沙漠 Solver 卡死）
- 现象：对拍/验证用的全状态 DP 或暴力枚举**按全规模实现**，状态数过亿（如状态 (t,p,w,f)≈3.6 亿），纯 Python OOM 卡死，求解阶段无限拖；本该"小规模对拍"的范式被当主解全量跑
- 为何扣分：不是逻辑错而是**工程不可行**——结果出不来=没结果；评委看不到数值，整篇坍塌
- 修复（把 PoC 冒烟闸从"逻辑可行"延伸到"性能可行"）：① 任何重型枚举/DP **动手前先估状态数/复杂度**；② 超阈值（经验 ~1e7 个状态）**强制粗化网格 / 缩短 horizon / 维度解耦 / 剪枝**；③ 先在**小规模实例冒烟**（秒级）确认正确再放大；④ "对拍/交叉验证"范式只需在**可解小规模**上与主解同值即可，不必全量。主解优先选有结构的精确法（如时空最长路+内层 LP），把指数级暴力留给小规模旁证

## D. 写作呈现类
**D1 图表非自洽**
- 现象：图没标题/轴标/单位/图例，正文不解读
- 修复：每张图能脱离正文看懂，且正文有文字解读

**D2 公式不编号 / 不规范**
- 修复：重要公式右编号，符号与符号表一致

**D3 通篇无小结、逻辑跳跃**
- 修复：每章末一句话承上启下

**D4 篇幅失衡**
- 现象：建模 10 页、结果半页
- 修复：结果与灵敏度是重点，别虎头蛇尾

## E. 致命级（直接掉档）
**E1 答非所问**：没回答题目明确要求的某一问 → Analyst 的子问题清单要逐条勾掉
**E2 抄袭/雷同**：模型与公开题解高度雷同 → 必须有自己的变体与创新命名
**E3 数据造假**：编造不存在的数据/结果 → 一票否决（这正是 Critic 反幻觉要守的底线）

## G. 美赛评委 triage 杀手（蒸馏自 paper/ 30篇官方点评 2015-2019；详见 `mcm_judge_commentary.md`）

> 这些是官方评委**反复点名、可直接淘汰**的失分点，Stage 9 终审 + Critic 上线前**逐条扫**。

### G1. 漏答题面子问题（**头号 triage 杀手**）
- 现象：多组件题只做一两问、或只做第一问耗光时间；答错变量（把给定量当未知去求）。
- 为何出局：评委先查"是否覆盖全部明示要求"，漏一个即排名暴跌，**即便你方法更高级**。
- 修复：建模前列全子问题清单，逐一打勾；先答全再深挖一面。

### G2. 致命缺陷违背常识却不回查（fatal flaw 一票否决）
- 现象：模型预测明显违反物理/常识（Newton 冷却写反），却不回头核。
- 修复：每个数值结果做 sanity check"这合理吗"；机理方向、量纲两侧守恒必查。

### G3. 建议/政策与模型脱节
- 现象：结论给"增加可再生能源/降 5%"这类凭空、不可执行的建议，未由模型结果导出。
- 修复：每条建议必须能指回某个模型输出；给**具体可执行步骤 + 数值**(多少/哪种/怎么做)。

### G4. 仿真/可视化与前文模型脱节（最常见 ICM 病）
- 现象：画了漂亮图/跑了仿真，但与前面建的模型对不上、各做各的。
- 修复：每张图都是某个前文模型的**实现**而非装饰；Our Work 配一张总框架流程图(Fig.1)串起全篇。

### G5. 正文必需结果丢进附录
- 现象：把关键结果/解释塞附录让评委自己翻，正文只剩方程。
- 修复：评委不该翻附录——重要结果与解读留正文，附录只放可运行代码/超长表。

### G6. 盗图盗公式不引用（视同抄袭，可取消资格）
- 现象：套用网络/课本的图、公式、模型不标来源（18/D 有 3% 队因此被取消资格）。
- 修复：正文 in-text 引用 + 文末双标注；用了现成模型必引并说明"我加了什么"。

### G7. 模型"建了不用"/过参数化做不了敏感性
- 现象：推了一堆方程却不用其结果；或模型过复杂到无法做敏感性分析。
- 修复：value-added 在现成模型上加东西并**用起来做决策**；宁可简单模型解释透，也别堆 8-10 个模型炫技。
