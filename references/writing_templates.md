# 写作模板 writing_templates

> 供 `modelcrew-writer`。竞赛"做得好不如写得清"，这份模板把高分论文的结构固化下来。

## 5 段式摘要模板（国赛，约 600–900 字）
> 摘要是评委 30 秒筛选的唯一依据，必须**最后写、基于定稿正文**。

1. **背景与问题**（1–2 句）：题目要解决什么，为什么重要。
2. **方法**（每个子问题一句）：针对问题 N，建立了【具体模型名 + 变体】，用【算法】求解。
   - 反例："我们用了优化模型" → 正例："针对配送问题，建立带时间窗的混合整数规划，用分支定界求解"
3. **关键结果**（**必须带数字**）：得到【量化结果】，误差/精度【数值】，较基线提升【数值】。
4. **创新点**（1–2 句）：本文的方法变体 / 跨学科组合 / 子问题间的复用。
5. **结论与意义**（1 句）：模型的适用性与推广价值。

→ 末尾：**关键词** 4–6 个（≥2 个具体方法 + ≥2 个题目领域词，禁止"数学建模/优化/分析"这类空词）。

## MCM/ICM Summary（1 页，250–350 词）
- 英文，单页，≥3 个量化结果。
- 结构：problem → approach → key results(with numbers) → takeaway。
- D/E/F 题另需一封 **Letter**：可操作建议 + 通俗语言 + 写明 caveat。

## 论文骨架（通用）
```
1. 问题重述与背景
2. 问题分析（每个子问题怎么想的）
3. 模型假设（每条带理由）
4. 符号说明（统一符号表）
5. 模型建立（分子问题）
6. 模型求解（算法 + 关键代码思路）
7. 结果与分析（图表 + 文字解读）
8. 灵敏度分析（独立大节，关键参数扰动）
9. 模型评价（优点 / 缺点 / 改进——把 Critic 的局限性如实写进来）
10. 结论
参考文献（GB/T 7714 或竞赛要求格式）
附录（完整代码）
```

## 措辞升级（让表述更有"设计感"）
| 平庸 | 升级 |
|---|---|
| 用了优化模型 | 建立考虑动态权重的多目标线性规划 |
| 结果不错 | 最优解较基线提升 23%，灵敏度区间内稳定 |
| 用 TOPSIS 评价 | 采用熵权-TOPSIS 组合赋权综合评价 |
| 预测了趋势 | 基于残差修正的 GM(1,1) 灰色预测，MAPE 4.1% |

> 铁律（呼应 Critic）：只写**经 Critic 放行或已标注不确定性**的结论，绝不把存疑结果写成定论。
