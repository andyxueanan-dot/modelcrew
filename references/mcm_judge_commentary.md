# 美赛官方评委口径 + O奖执行范式（蒸馏自 paper/ 真档案）

> **一手来源**：蒸馏自 `paper/`（用户本地 1.6GB 档案，**未入库**）——**30 篇官方 MCM/ICM Judge/Commentary 点评（2015–2019）** + **20 篇 Outstanding 团队论文（2022–2025，按题型抽样）**。
> 这是"评委到底奖励/惩罚什么"的**最高权威口径**；供 `modelcrew-judge`（打分锚点）、`modelcrew-writer`（写作目标）、`modelcrew-critic`（失分预警）查阅。每条尽量标 (年/题) 出处。
> ⚠️ 纯净基准注：已被抽样的 O 奖题（见文末清单）= 编排者"见过解"，刷题 iter 不得复用这些具体题。

## 1. 跨题型铁律（5 年评委反复点名，按"提及频次"排序）

1. **答全题面每个子问题 > 创新 > 沟通**：漏任一子任务/答错变量 → triage 直接出局，即便方法更简单也排名更低 (15/B, 16/D-E-F, 17/B-D, 18/A-B, 19/A)。**先答全所有明示要求，再深挖一面**。
2. **摘要必须含具体结果数字**（不只"打算做什么/用了什么方法"）——决定 triage 生死；"有结果无方法 > 有方法无结果" (15/all, 16/D, 17/B, 18/A, 19/D)。
3. **真做敏感性分析**（基准→逐个扰动参数→更高级：扰动假设）——**历年最弱项，缺则进不了终轮**；过参数化导致做不了敏感性也扣 (15/all, 16/A-C-E, 17/A, 18/A-B-C, 19/A-D)。
4. **沟通/写作是进决赛的最大判别器**：组织、段间过渡、文字+公式+图三者平衡；"写作把复杂讲清"≥高级数学 (15/C, 16/D-E-F, 17/D-E, 18/E, 19/D)。
5. **假设要带理由(justification)且真正用进模型**；常数给值+讨论误差；改假设若变结论须列为弱点；最强队把假设当敏感性来回访 (15/D, 16/C-D, 17/A, 18/A-D-F, 19/A-D-E)。
6. **引用！** 正文 in-text 引用；盗用网络/课本的图/公式/模型不标注 = 视同抄袭（18/D 有 3% 队因抄袭被取消资格）；文献综述要说清"我在前人基础上加了什么" (15/A-B-D, 16/B-C, 17/A-B-C, 18/A-D)。
7. **验证/校准**：拿历史案例(AF447)、已知数据、已发表指标对照；或删指标看输出是否变——把"好报告"升成"好论文" (15/B-C-D, 16/A-C, 17/C-E, 18/C-E, 19/E)。
8. **模型要"用起来"做决策、给可执行建议**，而非堆砌模型本身；**建议与模型脱节 = 失分**（"增加可再生能源"——多少？哪种？怎么做？）(16/A, 17/C, 18/C, 19/D)。
9. **value-added（在现成模型上加东西）**：让参数变函数、做改进(SEIR→SEIIR)、耦合多子模型——这是 Outstanding 的核心增量 (16/B, 19/A)。
10. **图必有标题/轴/单位/caption 且正文逐一解读**；图够大、标签清；重要结果留正文**不塞附录**（评委不该翻附录）(15/A-B-D, 16/A, 18/A-C, 19/A)。
11. **致命缺陷(fatal flaw)一票否决**：单个足够严重的机理错误可否决 O 奖（Newton 冷却写反差点丢奖）→ **结果违背常识必须回头查** (16/A)。
12. **简单但解释透 > 堆 8–10 个模型炫技**；选 1–2 个对路模型用好、配真实解读 (16/C, 17/D, 18/C-E)。

## 2. 致命失分清单（triage 秒杀，Critic 必查）

- 缺执行摘要 / 摘要写成 abstract、无问题或无结果 → 整篇出局 (16/D, 18/A)。
- 漏答题面某子任务（多组件题尤甚）→ 排名暴跌 (15/B 漏"不同搜索机型"、17/B 只做性能分析没做最优设计)。
- 列了现成模型却**不把给定数据接进模型** → triage 出局 (17/C)。
- 缺敏感性分析 → 进不了终轮 (15-18 全部)。
- 盗图/盗公式不引用 → 判抄袭 (18/A-D)。
- 建模/推方程却**不用结果**、把正文必需结果丢附录 (19/A)。
- 答错"非技术文体"（致航空公司的信写成新闻稿）(15/B)。
- 仿真/可视化与前文模型**脱节**（最常见 ICM 病）(19/D)。

## 3. 按题型得奖细则

- **A 连续/机理**：少变量 + **量纲两侧守恒** + 经典方程做骨架(Lotka-Volterra/Navier-Stokes/Archard 磨损律)再**改进**；**实验/曲线拟合标定参数**(测 Newton 冷却 k)；**解析互验 + 独立交叉验证**(理论磨深 4.871 vs 实测 4.5；3D 扫描独立佐证)；双查别违常识 (15/A, 16/A, 17/A, 18/A)。
- **B 离散/优化**：**明确目标函数**(最小时间/成本 或 最大发现概率)；**多目标同时权衡**(吞吐+安全+成本 全到，非取一二)；**Pareto Front 筛整数解**；最优性论证用「PSO 多次取均值 / 对照组比增长率 / 枚举全场景 / 结果落合理区间反验」；商业可行；敏感性**算弹性 s 并排序**指关键参数 (15/B, 17/B, 18/B, +O奖B)。
- **C 数据**：用**给定数据**（公平、免外部验证）；**缺失值插补非删行**；**降维后解释新变量含义**；**预测带不确定性区间**(C 区别 A/B 的核心)；训练/测试回测 + 加高斯噪声测稳健(准确率仅降 8%)；显式算 **ROI**；6 项评分(数据探索/画像/预测不确定性/建议/写作/可信) (16/C, 17/C, 18/C)。
- **ICM D/E/F**：题面**故意含糊**→先自定义输入(自查卢浮宫尺寸)；**强制面向决策者的政策/程序建议且由模型结果支撑**；跨学科 + 优秀文字(ICM 独立加分项)；**动态/随时间演化**受奖；评价型用**三级指标树 + Entropy/AHP/CRITIC 赋权 + Weighted TOPSIS/网络中心性排序**；网络建模(节点=实体/边=相关或流量/中心性)；**framing**(每模型绑真实决策者+真实地点)；**Letter/Memo 独立成章**面向非技术读者 (15/C-D, 16/D-E-F, 17/D-E, 18/D-E-F, 19/D-E-F)。

## 4. Outstanding 论文执行范式（蒸馏自 20 篇 2022–2025 O 奖，跨题型共性）

- **摘要**：首句点破现象/张力 → 逐任务分段("For Task k / Model I-III")，每段**粗体模型名 + 粗体硬数字结论**(relative error 2.16%、封顶 13,993 人、累积找回概率 0.5504)，结尾必带**稳健性 + 可推广性**一句。关键词只列方法不列对象。
- **章节高度同构**（O 奖几乎统一模板）：`Intro(Background→Restatement→Our Work+总框架流程图Fig.1) → Assumptions&Justifications(每条配理由) → Notations 表 → 分 Model 章(建模引入→变量→目标→约束→求解→结果解释) → Sensitivity → Strengths&Weaknesses → Letter/Memo → References → Appendix(可运行代码)`。
- **建模→求解→验证流水线**：拟合数据定参 → 机理/规划/评分网络建模 → 调库求解(PSO/GA/模拟退火/有限差分/Monte Carlo) → 指标标准化([0,1]/sigmoid) → 可视化 → 回测/对照/加噪声验证。
- **创新落点 = 命名 + 自造指标**：给现象/流程起**隐喻式标题 + 自创模型名**("Dual-Temporal Bayesian Network""Temporal Champagne Tower""Veils of Uncertainty""Rule Curves""Great Coach Effect")，并自造**可解释复合指标**(DPI=0.28·PE+0.32·PS+0.4·SA)。把成熟方法包装成专有命名——这是创新维拿分的最省力招。
- **存在性/因果用检验**：Ljung-Box + Runs Test 检随机性；DTW 比对模拟与真实反证；Hurdle+Tobit+2SLS 管零膨胀/删失/因果；贝叶斯变点检测识别"效应"。
- **可视化套路**：总框架流程图(必备 Fig.1)、相关/风险热力图、Pareto 3D 散点、Sankey 流向、网络图、甘特图(多年计划)、信息图式开篇。
- **模型组合 > 单模型**：一题多模型分工(GRU 预测 + RF 分类 + K-Means++ 聚类)，并与 XGBoost/RF 基线对照证更优。

## 5. 接入方式
- **Judge** 打分时按 §1 铁律 + §2 致命清单作负向触发器，按 §3/§4 作奖励锚点。
- **Writer** 按 §4 模板写（摘要逐任务+硬数字、Fig.1 框架图、Letter/Memo、命名创新）。
- **Critic** 用 §2 做上线前预警（漏子问题/无敏感性/建议脱节/盗图不引/结果进附录/违常识）。
