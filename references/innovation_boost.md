# 创新增压清单 innovation_boost

> 供 `modelcrew-modeler`（选完 baseline 后**强制过一遍**）与 `modelcrew-judge`（评创新性有锚点）。
> 把 `winning_paper_patterns.md` §3A-8 那句"创新落点 = 切入角度/组合/可解释命名 + 起可引用的名字"**展开成可执行的升档动作**。
>
> **核心主张**：创新分低，往往不是方法不够花，而是**停在了 baseline 组合、且没给它一个可被引用的名字**。
> 升档**不等于上更复杂的模型**——`winning_paper_patterns.md` §3A-1 明确"炫技复杂模型不结合实际=典型被刷"。
> 升档 = 在**可解释性不降**的前提下，沿下面三条路至少升一档，并**命名 + 在摘要/模型评价里显式标注**。

## 何时启动（Modeler 强制过闸）

选完每个子问题的 baseline 方法后，**逐题问一遍**："这个 baseline 能不能沿三条路之一升一档？升了是否仍可解释、仍服务题意？"
- 能 → 升档并命名，写进待审计结论清单（标注"创新点"）。
- 不能（升档会牺牲可解释性或脱离题意）→ **明确放弃并说明理由**（守住"简单可解释 > 炫技"，本身是 §2D Humboldt 范式的加分项）。

> 反面教材：`cases/demo_dispatch_simpson/` 初版三法（标准化/MH-OR/logistic）都是成熟工具的直接组合，创新只落在命名+类比，Judge 给创新维 7/10（最低）。升档示范：把"未达显著"用 E-value 量化因果稳健性（见 `inconclusive_playbook.md` 招 3）——这是**沿"问题操作化"路升了一档的真方法增量**，不是换个花模型。

## 三条升档路（任选其一即可，能叠加更好）

### 路 1 · 模型变体（baseline → 有动机的改进版）
给 baseline 加一个**针对本题痛点**的改进，不是为复杂而复杂。
| baseline | 升一档（按题意选） | 痛点动机 |
|---|---|---|
| 标准回归 | 岭/Lasso/分位数回归、加交互项 | 多重共线 / 异常值 / 关注分布尾部 |
| TOPSIS / AHP | 熵权-TOPSIS、组合赋权（主客观）、灰色-TOPSIS | 纯主观权重不服众 |
| ARIMA | SARIMAX（带外生变量）、残差修正 GM(1,1) | 有季节/外生驱动 / 小样本 |
| K-means | K-means++ + 轮廓系数定 k、迁移到子群再建模 | 初值敏感 / 分群后精度提升 |
| LP / IP | 多目标加权 / ε-约束 / 鲁棒优化 | 多冲突目标 / 参数不确定 |
| 单点估计 | + power 反算 / TOST / E-value（见 inconclusive_playbook） | 效应弱/不显著时把"做不出"升为决策价值 |

### 路 2 · 模型组合 / 流水线（两个简单模型串成一条有逻辑的链）
"先 A 后 B"且**A 的输出是 B 的有意义输入**，比单一模型更有设计感。
- 聚类 → 分群建模（K-means 找子群 → 各群单独回归/预测，迁移学习提精度，2406324 范式）。
- 降维 → 建模（PCA/特征选择 → 喂给分类器，附特征重要性/SHAP 归因）。
- 描述→质疑→校正三层链（如 Simpson 题：合并率陷阱 → 分层证伪 → 三法校正）。
- 机理 + 数据混合（物理/微分方程给结构，数据标定参数）。

### 路 3 · 问题操作化 / 跨学科类比（把模糊概念变成可计算量，借别学科成熟工具）
最高级、最难被复制的创新——**重新定义问题本身**。
- 把"公平/动量/韧性/拥堵"等模糊词**操作化**为一个可算的指标，并说清为什么这样定义。
- 借成熟学科工具迁移：流行病学人口标准化 → 调度公平比较；金融 VaR → 风险题；信息论熵 → 多样性/赋权。
- 把定性结论**量化**：不显著 → E-value 量化因果稳健性；"还需更多数据" → power 反算出具体 N。

## 命名范式（升完档必做：起一个可被引用的名字）

没名字的创新等于没创新——评委记不住、引用不了。命名公式：

`【修饰词：改进点/痛点/学科】 + 【方法核心】 [+ 比较法/模型/框架]`

| 平庸 | 命名后（可引用） |
|---|---|
| 两比例检验 | **难度配平比较法**（severity-matched fair comparison） |
| 加权 LP | 考虑动态权重的**多目标鲁棒选址模型** |
| LSTM | **注意力增强 LSTM + SHAP 归因**框架 |
| 分层+标准化+回归 | **"描述→质疑→校正"三层混杂校正框架** |

> 起名后必须在**摘要的创新点段**和**模型评价**里显式点出（§2B 国赛摘要末段、§2A 美赛创新句）。

## 实战复盘：创新维反复差一口的根因与最省力补法（蒸馏自刷题）

跨案例观测（`cases/practice_log.md`）：ModelCrew 的严谨/诚实/可复现稳定在国一水准，但**创新维反复卡 7/10**（demo_dispatch、practice_01 皆然），成为冲 90/国一的唯一系统性短板。根因不是方法不够，而是**把已经做出来的好东西"埋"了、没命名、没操作化成可报告的量**。最省力的 +1 补法（建模阶段就做，别等定稿）：

1. **给整套流程起一个可引用的名**（不是给某个算法）：如"留出竞赛选型 + 阻尼外推"、"难度配平比较法"、"描述→质疑→校正三层框架"。评委记得住才给得了创新分。
2. **把一个稳健性/等价/边界发现操作化成一个可报告指标**——这是把"诚实"转成"创新"的关键转化：
   - 两模型实质等价 → 报一个"**模型等价稳健性**"度量（指标差 / 预测重合度），明说"等价本身是稳健信号"。
   - 效应未达显著 → E-value / 达可判定所需 N（见 `inconclusive_playbook.md`）。
   - 结果对参数不敏感 → 一个"稳健性半径"或权重漂移幅度。
3. **一句话点出"别人会怎么做错、我这样做对在哪"**：把方法选择讲成有对手的判断（基线对照 / 反转判定准则 / 平局不宣布赢家），创新即"判断力"而非"算法花哨"。

> 一句话：**创新 = 把你已经做对的稳健性/诚实，命名 + 量化成一个评委记得住、引得了的东西。** 别让它沉默地躺在求解里。

## 给 Judge 的评分锚点

评创新维（国赛 10% / 美赛 looking-for"建模创造性"）时，对照本清单看：
- 论文是否沿三条路**至少升了一档**、且**起了可引用的名字**并在摘要/评价里标注？三者齐全 → 创新维可给 8+。
- 只有 baseline 组合、无升档、无命名 → 创新维 6–7（如 demo 初版）。
- 升档但脱离题意/牺牲可解释性炫技 → **不加反扣**（§5 雷区 4「套模型不结合实际」）。
- 守"简单可解释"且说明了为何不升档（如 Humboldt 刻意不合并指标）→ **不因"简单"扣创新分**，可正常给分。

## 命名变体速查库（蒸馏自 `handsomeZR-netizen/mathmodel-skill` distilled_naming，来源注明）

> 创新维最省力的一脚：别把 textbook 名裸写成标题（`AHP`/`BP`/`灰色预测`/`最短路` 都只是部件）。构词骨架 = **修饰词 + 核心方法 + 约束/对象 + 任务词**；修饰词从 {改进/动态/自适应/混合/复合/分阶段/鲁棒/基于…的/考虑…的} 取≥1。每个命名要能回答"改了什么·用什么改·针对什么场景"。

| 题型 | 现成复合命名（仿写锚点） |
|---|---|
| 优化/调度/规划 | 基于 Lagrangian 松弛的混合整数线性规划模型 · 考虑动态约束的多目标 MILP 鲁棒优化模型 · 结合启发式搜索与整数规划的分阶段调度模型 |
| 预测/时序 | 残差修正的 GM(1,1)-Markov 组合预测模型 · 考虑季节性扰动的 SARIMA-Prophet 混合预测模型 · 动态权重 LSTM-Attention 时序预测模型 |
| 评价/排序 | 动态权重 AHP-熵权综合评价模型 · 考虑指标相关性的改进熵权-TOPSIS 评价模型 · 灰色关联-熵权双重赋权评价模型 |
| 分类/识别 | 基于代价敏感学习的改进随机森林分类模型 · 引入注意力机制的多特征融合识别模型 · 考虑类别不平衡的集成学习分类模型 |
| 仿真/动力系统 | 考虑时变参数的改进 SEIR 动力学模型 · 基于事件驱动机制的离散事件仿真模型 · 考虑多阶段状态切换的马尔可夫链仿真模型 |
| 图论/网络 | 考虑时间窗约束的鲁棒 VRPTW 路径优化模型 · 基于最小费用流的分层网络调度模型 · 考虑容量瓶颈的多阶段网络分配模型 |
| 统计/融合/因果 | 基于反事实推理的数据融合评价模型 · 考虑异方差结构的稳健回归与区间估计模型 · 带先验约束的自适应因果效应估计模型 |

> 用法：命名最好能直接映射到正文的一个子问题，让评委在目录里就看见你的方法链。**注意**：命名是"如何表述已做对的工作"，不是凭空加方法——名实必须相符（否则触 `anti_patterns A3` 摘要与正文不符）。
