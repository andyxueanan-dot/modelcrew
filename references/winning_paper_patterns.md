> 从**美赛 MCM/ICM O 奖**与**国赛 CUMCM 国一**的官方评审材料 + 真实获奖论文中蒸馏出的**可执行规律**，

## 1. 评委如何评分

### 美赛 MCM/ICM（✅官方两阶段）
- **Triage 初筛**：每篇 **≥2 位评委**评阅按分排序；判官来自数十学科（不只数学家）。**初筛主要看摘要 + 整体组织/可读性**。⚠️"近一半论文仅凭摘要被淘汰"是建模指南广泛口径（COMAP 未给官方百分比）。
- **Final 终评**：高分论文进入，由约 29 位终评评委连续数天多轮当面讨论。
- **奖级**：Outstanding=四维（建模/求解/分析/沟通）都顶尖的"best of the best"；Finalist=进终评末轮；Meritorious=完整回应全部要求且多方面优秀；Honorable=高于平均、过程有据；Successful=完成提交。

### 国赛 CUMCM（✅官方两级 + 主标准）
- **赛区初评 → 全国复评**两级；评委**回避本校**；正式评阅前**试评 + 统一评阅细则**（评委带同一张细则锚点快速比对——**把得分点摆到评委找得到的地方**）。
- **✅四大评奖主标准（顶层 rubric，逐字记）**：**假设的合理性、建模的创造性、结果的正确性、表述的清晰程度。**
- **✅官方评阅要点的解题链条**（段落级 checklist，缺一环=结构性失分）：建模思路 → 建模方法 → 模型表示 → 求解方法与步骤 → 模型结果 → 结果分析与检验 → 模型评价与改进方向。
- **✅初筛**："全国评阅时将首先根据摘要和论文整体结构及概貌对论文优劣进行初步筛选"——**档次在前几分钟被摘要决定**。
- ⚠️国一率约 0.68%–0.9%、每题国一约 100 个、约 50 评委每人评 ~120 份百分制再标准化（社区口径，按年核对）。

## 2. 摘要黄金范式（**生死线**）

### 2A. 美赛英文摘要（约 250–350 词、≤1 页）
按序：① 一句点题 + 对象/数据 → ② **每个子问题：方法点名 + 具体结果(带数字)** → ③ **敢下判断**的关键结论 → ④ 敏感性/稳健性一句 → ⑤ 现实落地(memo/letter) → ⑥ Keywords(3–6 个方法/概念)。
- **DO**：结论态动词（proves/shows/we find）；点名模型算法；摘要里直接放数字和结论。
- **DON'T**：流水账（"先建模再仿真"）；制造悬念（"结论见正文"）；只说"做了什么"无结果；堆术语无数字 = triage 秒刷高危。

### 2B. 国赛中文摘要（含标题关键词 ≤1 页、⚠️约 800–1100 字、**不放图表**）
按序：① 开头 1–2 句总览（针对什么问题、做了什么）→ ② **按问题分段**，每段写死四件事：**模型名称 / 求解算法名称 / 关键结果的具体数值 / 现实意义一句** → ③ 结尾：创新点 / 优点 / 推广性收口 → ④ 关键词 3–5 个（写**问题与核心方法术语**，⚠️只写软件名 MATLAB/SPSS = 低级失误）。
- **DO**：每段以"针对问题X，本文…"开头让评委一眼定位；优化/预测/物理类**摘要必须出现数值**。
- **DON'T**：摘要只讲方法不报结果（**无数值=直接降档**）；写成问题重述/文献综述；空话；超一页；放图表。

### 2C. 摘要自检 5 问（写完逐条核对，Writer 必跑）
解决了什么问题？/ 用了什么模型（点名）？/ 怎么求解（算法名）？/ **结果数值是多少**？/ 有无敏感性与改进方向？

### 2D. 标杆对照（真实 2024 MCM-C O 奖摘要 队号 2406324，可当 Writer 仿写锚点 / Judge 对照标杆）
其胜点：开篇锁定"对象+数据"；**每子任务=方法点名+明确结论**（EWMA 定义动量→run test "**proves momentum is not random**"→LSTM+XGBoost 预测→K-means 迁移学习提精度→SHAP 找关键因子）；**主动暴露弱点**（换女子数据误差变大并解释）；含敏感性（验证 step size、cluster number）；给教练写 memo；~300 词全是方法+结论零废话。
> 另一反直觉范式（2015 ICM-E O 奖 Humboldt）：六域各做**线性加权指标 + radar 图**，**刻意不合并成单一指标**被评委点赞——**简单可解释 > 炫技**。

## 3. 建模 / 求解 / 可视化得奖共性（每条可执行）

### 3A. 跨题型通用（✅多数源自 COMAP looking-for / CUMCM 主标准）
1. **由简到繁**：先给能跑通能解释的 baseline（SIR/AHP/回归/LP…）拿到结果，再做**有动机的改进**。**一上来炫技复杂模型不结合实际 = 典型被刷**。
2. **每个选择都配"为什么"**（模型/参数/假设的理由都写出——美赛 "justification for all choices" 官方原话）。
3. **假设"被论证 + 必需"**：单设 Assumptions 章；每条给动机（最好引文献"前人也这么假设"）；只写对解法必需的，别堆无关假设。
4. **敏感性 / 稳健性分析 = 分水岭**：扰动关键参数看结论稳不稳 + 误差分析。**无敏感性≈拿不到国一 / 缺美赛官方 looking-for**。
5. **模型验证（verification）**：held-out / 换数据集 / 与现实对照证明不是过拟合（2406324 换女子数据并诚实报告误差变大）。
6. **结论 → 现实建议闭环**：建议必须由模型结果导出；题目要 memo/letter 就一定写（**不得署真名**）。
7. **一图一结论 + 必有文字解读**：图表无解读=失分；国赛中文规范——**三线表、表题在上图题在下、图表都编号、正文引用"如图X"**。
8. **创新落点**：不是用最花的算法，而是**问题切入角度 / 模型组合 / 可解释性**；并**给创新起可被引用的名字**（"两阶段优化模型""改进的 SARIMAX"）在摘要与模型评价里显式标注。

### 3B. 按题型（A 机理 / B 优化 / C 数据）
- **A 连续/机理**：机理建模(微分方程/动力系统/物理推导) + 数值求解(说明算法/步长/收敛) + 与物理直觉或实测对照；加分：量纲检查、边界/极端情形、解稳定性、参数标定来源可靠。
- **B 优化**：问题转化 → 决策变量/约束/目标函数清晰列出 → 选对模型(LP/IP/NLP/DP/图论) → 求解器或启发式(SA/GA/PSO)求解 → 报最优值 → 灵敏度；加分：最优性论证或 gap、多场景对比；**启发式必报参数与随机种子（否则不可复现）**。
- **C 数据**（国奖深度门槛）：数据清洗与质量诊断扎实并写进论文 → 特征工程有思路 → 模型与题意匹配（时序 ARIMA/SARIMAX/Prophet、聚类 K-means++…，先统计分析建立认识再上模型）→ 可视化充分 + **每图业务解读** → 灵敏度 + 误差评估（交叉验证/残差）；**别把相关当因果、别"调包出图"不结合业务**。

## 4. 格式 / 诚信硬规范（违规可直接降级或取消资格）

### 美赛（✅COMAP）
≤25 页（目录/参考/附录/AI 报告除外）；每页页眉 `Team # / Page x of y`；**全文不得出现姓名/学校/地区**；字号 ≥12pt；PDF <20MB；**摘要必须是 PDF 第一页**；用了 AI 须附 **AI 使用报告**（不计入 25 页）；数据/图片不标来源可降级或判抄袭。

### 国赛（✅CUMCM 格式规范）
- 页面顺序固定：**第1页承诺书 → 第2页编号页(选手不填) → 第3页摘要页(题目+摘要+关键词 ≤1页，无需英文) → 第4页起正文(⚠️尽量≤20页、不要目录) → 附录(不限页)**。
- **页码从摘要页起编"1"**、页脚居中连续；**任何位置不得泄露身份**（含页眉），违者**无条件取消评奖资格**。
- 参考文献按科技论文规范，正文与文末双标注；网络资源标网址 + 访问时间。
- **附录须含完整可运行源程序**；缺必要源程序 / 程序跑不出论文结果 → **可能取消资格**。
- 电子版 PDF/Word ≤20MB（**不含承诺书与编号页**），第一页必须是摘要页。
- **✅查重红线（硬闸门）**：相似度 **≥25% 不能报送全国**；**>30% 不获成功参赛证书**；**>40% 作违纪处理**。**代码雷同也算抄袭**；用开源须实质改写并标注；引用须正文+参考文献双标注。

## 5. 失分雷区清单（triage 秒刷特征 → Critic/Judge 的负向触发器）
1. 摘要无结果/无数字、写成流水账或问题重述、制造悬念 → 第一轮被刷（近半死在这）。
2. 答非所问 / 漏做要求的 memo·letter / 改变问题范围。
3. 无敏感性分析 / 无模型验证。
4. 套模型不结合实际、过度复杂炫技、模型无法解释。
5. 假设堆砌、无论证、对解法非必需。
6. 建议凭空提、与模型结果无关。
7. 图表无解读 / 不编号 / 表题图题位置错。
8. 无 in-text 引用 / 数据图片不标来源。
9. 不可复现（启发式/随机算法不报参数与种子）。
10. **美赛**：正文泄身份、超 25 页、漏 AI 报告、PDF>20MB、页眉无队号。
11. **国赛**：缺承诺书/编号页、摘要超一页、有目录、正文失控、**查重超线、附录无可运行代码** → 取消资格风险。

## 6. looking-for → Judge rubric 锚点

### 美赛 ✅官方 looking-for（终评打分维度，逐条可当 rubric）
写得清楚好读 · 答到题目每个子问题 · **每个选择都有理由** · 假设**被论证且必需** · 模型详解**且为何合适** · **模型验证** · **敏感性分析** · **建议由结果导出** · 优缺点明写 · **强引用 + in-text 引用**。

### 国赛 ✅四大主标准（顶层四维 + §1 七环节解题链作子项）
**假设合理性 · 建模创造性 · 结果正确性 · 表述清晰度。**
> ⚠️坊间流传的"可行性20/算法20/创新10/规范10/潜力20/可操作20"逐分权重**未经官方逐条确认**，喂 Judge 时只作"参考权重"，别当官方 rubric。

### 给 Critic 的一条铁律（直接来自官方 looking-for）
结论上线前必须能回答四问：**敏感性做了吗？模型验证了吗？建议是从结果导出的吗？假设是必需且被论证的吗？** 答不上即 ⚠️/❌。
